# R7 Operator Audit

> 状态：审名 v0.1 (2026-05-13)。
> 范围：R7 因的 128 个 operator reader names；R6 见 [R6-operator-audit.md](R6-operator-audit.md)。
> 形式口径：每个 bitstring `m` 表示 `(Z/2)^7` 上的平移 `T_m(s)=m xor s`。若 `e_i` 是第 `i` 位 basis，则 `τ_i = T_{e_i}`，所有 `τ_i` 交换。

## Naming Rule

- 精确名永远是 `T_m` 或 `τ_i` 的和成；中文名只是 reader alias。
- R7 沿用 R6 的成词收束：`變化 = 化+變`，`改變 = 改+變`。
- 若 `化+變` 与 `改` 同时出现，优先写 `改+變化`，避免把 y2 吸进 `改變`。
- 第七位原子读作 `印`：它是 operator 的留痕/登记动作；cell 的状态仍读 `無跡/有跡`。
- 其余多原子组合用 `+` 分隔，不写成 `改化變臨易內易外印` 这种串。
- `儒道法佛 / 形式逻辑数学 / 哲学 / 科学` 四列是跨语境表达候选；不是 parser 名，也不是声称各传统已有这些 operator 的严格定名。

## Notes

- `negate y_i` 指 coordinate negation，不是整句命题否定；多个 y 位表示同时翻转对应坐标。
- `印` 只给第七位 operator 命名；R7 cell 的可读状态仍是 `無跡/有跡`，避免把 state 和 operator 混作一个词。
- 每个非零 `T_m` 都是 involution，但 `involution` 只说明自反性，不能区分是哪一个 mask，所以不能作为公开名。
