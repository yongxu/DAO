# 归档政策

归档政策用于处理旧文、史料、过时草稿和被新版替代的说明。目标是保留来源，同时避免旧口径继续冒充当前状态。

## 状态

| 状态 | 含义 | 处理 |
|---|---|---|
| live | 当前维护入口 | 保持链接和生成索引同步 |
| indexed | 旧文仍有用，已由新版索引 | 不改原文，在新版写读法 |
| frozen | 历史快照 | 不修辞、不合并、不删减 |
| superseded | 被新版替代 | 标注替代入口 |
| pending | 待落地接口或草稿 | 不升级为结论 |

## 史料目录

`史料/` 默认视为 frozen。除专门归档任务外，不在其中修改文字。新版需要引用时：

- 使用短路径指针。
- 说明当前读法。
- 避免长段摘录。
- 若旧文含过强表述，在新版中降级为 claim 或 historical source。

## 旧义理目录

`义理/` 是论证来源，不是新版最终目录。迁移时优先做：

- 归类到 `../20_theory_义理/`。
- 互证到 Lean、claim 或 generated index。
- 把强主张拆成可审计状态。
- 保留旧文标题和路径，方便回查。

## 废弃与替代

发现旧页已被替代时，不直接删除。先在新版对应页写：

- 旧入口。
- 新入口。
- 替代范围。
- 尚未替代的残余内容。

删除只应发生在明确清理任务中，并确认没有索引、链接或史料依赖。

## v14 与骨架稿

`史料/生生不息论_v14_形式证明骨架版.md` 属于历史骨架。当前读法应回到 Lean、`../_generated/` 和维护页。若骨架稿中的目标尚未完成，应写为 roadmap 或 pending，不写为已达成。
