# 文档契约

docs-next 的契约是：人工页解释口径，生成页反映机器事实，旧史料保持冻结。维护时先判断一个修改属于哪一层。

## 层级

| 层 | 位置 | 维护方式 |
|---|---|---|
| 形式事实 | `formal/`、Lean theorem、build 结果 | 由 Lean 和脚本产生 |
| 生成索引 | `docs-next/_generated/` | 由脚本生成，不手改 |
| 新版人工文档 | `docs-next/` 各分区 | 摘要、导航、状态说明 |
| 旧义理 | `义理/` | 保留来源，逐步由新版索引 |
| 史料 | `史料/` | frozen snapshot，除归档工作外不改 |

## 人工页必须做到

- 指向事实来源，不替代事实来源。
- 区分 theorem、claim、model、registry、pending、archive。
- 涉及统计时引用 `../_generated/`，不手写易过期数字。
- 避免长引旧文。
- 对跨域论断保留条件和边界。

## 人工页不得做到

- 不把 pending 写成 proved。
- 不把 ledgerDependent 写成 machineChecked。
- 不把图或表写成证明。
- 不把 old draft 的强表述搬进新版而不降级。
- 不编辑生成文件来修正文档叙述。

## 冲突处理

| 冲突 | 处理 |
|---|---|
| 人工页与生成索引冲突 | 先信任生成索引，核对源文件，再改人工页 |
| 生成索引与 Lean 直读冲突 | 重新运行生成流程；若仍冲突，检查脚本 |
| 义理旧文与 Lean 冲突 | 新版标注当前形式状态，不改旧文 |
| 史料与新版术语冲突 | 保留史料原貌，在新版写迁移读法 |

## 合作规则

多人并行时，只改自己拥有的分区。发现他人已改文件，不回滚；若同页必须改，先读现状并做最小追加。
