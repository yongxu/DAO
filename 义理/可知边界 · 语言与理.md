>
> **语言的边界，就是可描述之理的边界。**
>
> 在此之内，可说、可理、可知三者同界；在此之外，不可说。若仍强行说出，也只是一个话语事件，不再是理。

## 〇 · 边界声明

本文只处理“可知”的边界，不处理全部“可”的边界。

| 边界 | 本文处理吗 | 说明 |
|---|---|---|
| 可知 / 不可知 | 是 | 以语言可描述之理为边界 |
| 可说 / 不可说 | 是 | 可说 iff 可描述为理 |
| 是理 / 非理 | 是 | 边界之外说了也非理 |
| 可感 / 不可感 | 另文处理 | 见 [可的边界 · 感行证](可的边界%20·%20感行证.md) |
| 可行 / 不可行 | 另文处理 | 见 [可的边界 · 感行证](可的边界%20·%20感行证.md) |
| 可证 / 不可证 | 另文处理 | 见 [可的边界 · 感行证](可的边界%20·%20感行证.md)，只作 typed skeleton |

所以本文不是说“不可说之外什么都没有”。更准确是：

> 不可说之外，不在“语言可描述之理”的边界内；它可能属于可感、不可感、可行、不可行、可现、不可现等其他边界。其中可感 / 可行 / 可证三组已由 [可的边界 · 感行证](可的边界%20·%20感行证.md) 承接。

## 一 · 为什么语言边界就是理边界

语言若只是声音、字形、符号串，它可以无限噪声化。任何东西都可以被发声、被写下、被拼接。

但这不是本文所说的“语言”。

本文所说语言，是能进入可知结构的语言：

| 层 | 说明 |
|---|---|
| 字符串 | 可以任意生成，不保证有理 |
| 句子 | 有局部语法，不保证可审计 |
| 论述 | 有指向，但可能仍含混 |
| 可描述之理 | 可被归入定义、边界、推理、审计 |

因此，语言的有效边界不是“能不能发出声音”，而是：

```
能否描述为理
```

也就是：

```
language boundary = describable Li boundary
```

Lean 中：

```lean
abbrev LanguageExpression := WenExpression
abbrev DescribableLiExpression := LiExpression

def languageBoundary (w : LanguageExpression) : BoundaryKind :=
  wenLimit w

def describableLiBoundary (l : DescribableLiExpression) : BoundaryKind :=
  liLimit l
```

上一层已经证明：

```lean
liLimit l = wenLimit w
```

所以本文直接得到：

```lean
theorem language_boundary_eq_describable_li_boundary
    (w : LanguageExpression) (l : DescribableLiExpression) :
    languageBoundary w = describableLiBoundary l
```

这就是主 claim 的形式版。

## 二 · 可说、是理、可知三者同界

本文只在“可知”范围内说话。

可知不是所有存在的总和；可知是已经进入语言可描述之理的范围。

Lean 中把区域压成两类：

```lean
inductive KnowableRegion : Type
  | withinDescribableLi
  | beyondDescribableLi
```

然后定义：

```lean
def Sayable (r : KnowableRegion) : Prop :=
  r = .withinDescribableLi

def IsLi (r : KnowableRegion) : Prop :=
  r = .withinDescribableLi

def Knowable (r : KnowableRegion) : Prop :=
  IsLi r
```

于是：

```lean
theorem sayable_iff_is_li (r : KnowableRegion) :
    Sayable r ↔ IsLi r

theorem knowable_iff_sayable (r : KnowableRegion) :
    Knowable r ↔ Sayable r
```

读法：

| 命题 | 含义 |
|---|---|
| 可说 iff 是理 | 有效语言必须在可描述之理内 |
| 可知 iff 可说 | 本文的可知边界由语言可描述性刻画 |
| 可知 iff 是理 | 可知不是心理感觉，而是可描述、可审计、可归理 |

这不是把世界缩成语言；这是限定本文讨论的“可知”层。

## 三 · 边界之外，不可说

边界之外并不是“没有”，而是“不在此边界内”。

在可知边界中，边界之外就是：

```lean
.beyondDescribableLi
```

可证明：

```lean
theorem beyond_describable_li_is_unsayable :
    ¬ Sayable .beyondDescribableLi

theorem beyond_describable_li_is_not_li :
    ¬ IsLi .beyondDescribableLi
```

这就是：

```
在此之外，不可说；
在此之外，不是理。
```

注意，这里的“不可说”不是禁止发声，也不是禁止写字。

它的意思是：

> 若某内容超出可描述之理的边界，它就不能作为理性语言被说出。

---

## 四 · 说了也非理

人当然可以说越界的话。

比如：

```
X 超出任何可描述之理，但我现在完整描述了 X。
```

这句话作为字符串存在；作为话语事件也存在。但它不再是理。

Lean 中建一个最薄的 `Utterance`：

```lean
structure Utterance where
  region : KnowableRegion
```

如果一个话语落在界外：

```lean
def BeyondBoundaryUtterance (u : Utterance) : Prop :=
  u.region = .beyondDescribableLi
```

那么它不是理性话语：

```lean
theorem beyond_boundary_utterance_is_non_li
    (u : Utterance) (h : BeyondBoundaryUtterance u) :
    ¬ IsLiUtterance u
```

这给出“说了也非理”的形式化版本。

换句话说：

| 状态 | 是否存在 | 是否是理 |
|---|---|---|
| 界内表达 | 存在 | 是理 |
| 界外沉默 | 不作为语言出现 | 不进入理判定 |
| 界外话语 | 可作为 utterance 出现 | 非理 |

---

## 五 · 这是可知的边界

本文的“可知”不是所有“可”的总边界。

它只回答：

```
什么能作为语言中的理被知道？
```

答案：

```
能落在可描述之理边界内的东西。
```

所以：

```
可知边界 = 可说边界 = 可描述之理边界
```

主定理合成如下：

```lean
theorem knowable_boundary_summary :
    (∀ w : LanguageExpression, ∀ l : DescribableLiExpression,
      languageBoundary w = describableLiBoundary l)
    ∧ (∀ r : KnowableRegion, Sayable r ↔ IsLi r)
    ∧ (∀ r : KnowableRegion, Knowable r ↔ Sayable r)
    ∧ ¬ Sayable .beyondDescribableLi
    ∧ ¬ IsLi .beyondDescribableLi
    ∧ (∀ u : Utterance, BeyondBoundaryUtterance u → ¬ IsLiUtterance u)
```

读法：

1. 语言边界就是可描述之理的边界；
2. 可说 iff 是理；
3. 可知 iff 可说；
4. 界外不可说；
5. 界外非理；
6. 界外若被说出，也非理。

## 六 · 可感 / 可行 / 可证的承接

边界之外还有别的“可”。

例如：

| 边界 | 问题 |
|---|---|
| 可感 / 不可感 | 能否被感受、感知、现象化 |
| 可行 / 不可行 | 能否被实践、执行、落实 |
| 可证 / 不可证 | 能否在某形式系统内证明 |
| 可测 / 不可测 | 能否经测量接口落成结果 |
| 可名 / 不可名 | 能否被命名但不必被证明 |

这些不等于可知边界。

本文只在 Lean 中留下留项锚点，表示这些不属于本文的可知边界：

```lean
inductive DeferredKeBoundary : Type
  | feelable
  | unfeelable
  | actionable
  | unactionable
  | provable
  | unprovable

theorem deferred_ke_boundaries_are_not_handled_here
    (b : DeferredKeBoundary) :
    handledHere b = false
```

其中可感 / 可行 / 可证三组已经由 `KeBoundary.lean` 承接：

```lean
theorem deferred_ke_boundaries_handled_here (b : DeferredKeBoundary) :
    handledHere b = false ∧ handledInKeBoundary b = true
```

所以这里的意思不是“项目未处理”，而是“可知边界本文不处理；另文按感、行、证三轴处理”。后续若继续推进可测 / 可名，应同样分别建立它们与可知边界的关系，而不是默认全部并入语言边界。

---

## 七 · 结论

本文的结论可以压成一句：

> 语言的边界，是可描述之理的边界；在此之内，可说、可知、可理同界；在此之外，不可作为理说出，说了也非理。

这给后续“可”的总边界留下清晰入口：

```
可知：语言可描述之理
可感：见《可的边界 · 感行证》
不可感：见《可的边界 · 感行证》
可行：见《可的边界 · 感行证》
不可行：见《可的边界 · 感行证》
可证：见《可的边界 · 感行证》
不可证：见《可的边界 · 感行证》
可测：未纳入本轮
可名：未纳入本轮
```

下一步如果继续推进，不应再问“边界外是否存在”，而应问：

```
它属于哪一种可？
```
