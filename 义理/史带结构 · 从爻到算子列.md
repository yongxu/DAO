## Claim

> 史可读作带。带上存 192 格；每格含卦与时。卦可展开为六爻，爻即元。
> 结构可由算子列递归生成：或为算子列作用于元，或为算子列作用于两个已生成结构。
> 算子列在此接口中承担“分”的角色：分辨、分化、展开结构。完整名在此接口中承担“和”的角色：把所指、符号、约定、域、消歧合束成可名接口。
> 因此本轮能证明的是：在当前形式层内，史带、爻根、算子列递归形状可闭合。

## 边界声明

这里证明的是 typed skeleton，不是把所有历史、所有文义、所有算子语义一次证明完。

- `HistoryTape` 是 `List Cell256`，是无界列表/栈式带；它不是双向无限图灵机纸带。(v3 重号：原 `List Cell192` 已迁移至 Cell256 = Hexagram × V₄ Klein Shi。)
- `StructureExpr` 记录“算子列递归形状”；371 个文言算子已有保守机器 denotation，但不声称都已有完整领域专义。
- “算子皆分”不是说所有算子字面等于 `fenToYi` / `fenToTrigram`；而是说在本接口中，算子列的结构角色是分辨与分化。
- “名皆和”不是说名字保证名实完全相符；而是说完整名把五个命名轴合束为可名接口。
- `乾坤，一，元，爻` 现在只读作根层命题：`一` 是唯一根，`元 = 爻`，乾坤是阳/阴二极及其三爻呈现；`乾坤之间` 在本文只读作上中下三层之间，不建立 endpoint `interval` 对象。

## 更正：乾坤之间即三层之间

“乾坤之间”这个说法不该收回；应收回的是 endpoint interval 读法。更精确地说：

- 这里要表达的不是“乾与坤之间有一个端点区间对象”。
- 这里要表达的是：乾坤即阳/阴二极；二极一对照，就显出三爻的下/中/上。
- Lean 中对应为 `RootOne : Unit`、`Yuan = Yao`、`qianKunYuanPoles = (yang, yin)`、`qianKunTrigrams = (qian, kun)`。
- `间` 在此处对应 `qianKunJianReading = .betweenLayers`，不是 `.endpointInterval`。
- 语言层已有 `ContextCue.betweenNominals` / `ContextCue.betweenActions`，所以“间”有 between 读法；本文把这个读法落到三爻层。
- 泛型 interval 口径仍可在 `Pending/Interfaces.lean` 或物-间-事层讨论，但本文只处理三层之间的 between-layers 读法。

## 上中下层系统

代码里有三种层级表达，不能混成一个。

1. 三爻内的上中下：`Trigram.y1/y2/y3` 分别是下/中/上；本文用 `VerticalLayer` 和 `layerYao` 明确化。
2. 六爻内的内外：`Hexagram.innerTrigram` 与 `outerTrigram` 把六爻拆成内卦/外卦；每个三爻内部仍有下/中/上。
3. 六爻位结构判断：`HexagramPosition.lean` 用 `centralYaos`、`respondsCount`、`biCount` 判断中爻、应位、比位等全卦关系。

因此，“上中下层之间如何表达”在本轮的最小答案是：

```text
间的读法：qianKunJianReading = betweenLayers
层集：qianKunJianLayers = [下, 中, 上]
判断：layerOfTrigramIndex
取值：layerYao
关系：layerRelation
分：fenAtLayer
合：removeLayer
```

核心闭合：

```lean
theorem removeLayer_fenAtLayer (l : VerticalLayer) (s : SiXiang) (y : Yuan) :
    removeLayer l (fenAtLayer l s y) = s
```

## 缺什么

强版本还缺四类定义。

1. 若要另写 `乾坤之间` 的路径/距离/拓扑/过程语义，那属于间论或物-间-事层，不属于本文的根层/层间 claim。
2. 全算子领域专义：371 个 `OperatorId × Cell256` 已有保守机器 denotation (v3：原 `Cell192`)；但许多仍是 signature carrier，不等于完整文本/领域语义。
3. 真正图灵纸带模拟：若要证明“史就是图灵机纸带”，需要双向 tape/head/transition，并给出从 TM 或 Minsky machine 到 `YiInstr` 的完整模拟定理。
4. 强名实理论：若要证明“名一定合于实”，需要对象语义、指称正确性与名实相符谓词。

theorem history_tape_structure_summary :
    (∀ s : YiState, stateTape s = s.history)
    ∧ (∀ s : YiState, stateTape (YiState.execute YiInstr.push s) = s.cur :: stateTape s)
    ∧ (∀ c : Cell256, (cellYuanTrace c).length = 6)  -- v3：原 `∀ c : Cell192`
