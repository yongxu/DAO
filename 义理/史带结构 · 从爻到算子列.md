## Claim

> 史可读作带。带上存 192 格；每格含卦与时。卦可展开为六爻，爻即元。
> 结构可由算子列递归生成：或为算子列作用于元，或为算子列作用于两个已生成结构。
> 算子列在此接口中承担“分”的角色：分辨、分化、展开结构。完整名在此接口中承担“和”的角色：把所指、符号、约定、域、消歧合束成可名接口。
> 因此本轮能证明的是：在当前形式层内，史带、爻根、算子列递归形状可闭合。

## 边界声明

这里证明的是 typed skeleton，不是把所有历史、所有文义、所有算子语义一次证明完。

- `HistoryTape` 是 `List Cell192`，是无界列表/栈式带；它不是双向无限图灵机纸带。
- `StructureExpr` 记录“算子列递归形状”；并不声称 371 个文言算子都已有可执行 denotation。
- “算子皆分”不是说所有算子字面等于 `fenToYi` / `fenToTrigram`；而是说在本接口中，算子列的结构角色是分辨与分化。
- “名皆和”不是说名字保证名实完全相符；而是说完整名把五个命名轴合束为可名接口。
- `乾坤之间` 现在只补成端点型 typed skeleton：`QianKunInterval.start = Trigram.qian` 且 `stop = Trigram.kun`。它不是度量空间、拓扑区间或物理时空区间。

## 上中下层系统

代码里有三种层级表达，不能混成一个。

1. 三爻内的上中下：`Trigram.y1/y2/y3` 分别是下/中/上；本文用 `VerticalLayer` 和 `layerYao` 明确化。
2. 六爻内的内外：`Hexagram.innerTrigram` 与 `outerTrigram` 把六爻拆成内卦/外卦；每个三爻内部仍有下/中/上。
3. 六爻位结构判断：`HexagramPosition.lean` 用 `centralYaos`、`respondsCount`、`biCount` 判断中爻、应位、比位等全卦关系。

因此，“上中下层之间如何表达”在本轮的最小答案是：

```text
判断：layerOfTrigramIndex
取值：layerYao
关系：layerRelation
分：fenAtLayer
合：removeLayer
```

核心闭合：

```lean
theorem removeLayer_fenAtLayer (l : VerticalLayer) (s : SiXiang) (y : Yuan) :
    removeLayer l (fenAtLayer l s y) = s
```

## 缺什么

强版本还缺四类定义。

1. 完整 `乾坤之间` 理论：本文只有端点 skeleton；若要更强，需要定义方向、路径、距离、拓扑或过程语义。
2. 全算子 denotation：371 个 `OperatorId` 目前有签名账本，但只有 L0 指令与一部分 `Cell192` 变换有机器语义。
3. 真正图灵纸带模拟：若要证明“史就是图灵机纸带”，需要双向 tape/head/transition，并给出从 TM 或 Minsky machine 到 `YiInstr` 的完整模拟定理。
4. 强名实理论：若要证明“名一定合于实”，需要对象语义、指称正确性与名实相符谓词。

## Lean 证明入口

```lean
theorem history_tape_structure_summary :
    (∀ s : YiState, stateTape s = s.history)
    ∧ (∀ s : YiState, stateTape (YiState.execute YiInstr.push s) = s.cur :: stateTape s)
    ∧ (∀ c : Cell192, (cellYuanTrace c).length = 6)
    ∧ (∀ y1 y2 y3 : Yuan,
        fenToTrigram (fenToSiXiang (fenToYi () y1) y2) y3 = ⟨y1, y2, y3⟩)
    ∧ (∀ e : StructureExpr, BuiltFromYao e)
    ∧ (∀ ops : CatalogueOperatorSeq, operatorSeqAct ops = .fen)
    ∧ (∀ m : MingSkeleton, nameAct m = .he)
    ∧ (∀ ops : CatalogueOperatorSeq, ∀ name : MingSkeleton, ∀ e : StructureExpr,
        CompleteMing name →
          operatorSeqAct ops = .fen
            ∧ nameAct name = .he
            ∧ WellFormedNamedStructure (namedByHe name (splitByOperators ops e)))
    ∧ qianKunInterval.start = Trigram.qian
    ∧ qianKunInterval.stop = Trigram.kun
    ∧ (∀ l : VerticalLayer, layerYao qianKunInterval.start l ≠ layerYao qianKunInterval.stop l)
    ∧ (∀ l : VerticalLayer, ∀ s : SiXiang, ∀ y : Yuan,
        removeLayer l (fenAtLayer l s y) = s)
    ∧ fullOperatorSignatures.length = 371
    ∧ l0InstructionClauseKinds.length = 12
```
