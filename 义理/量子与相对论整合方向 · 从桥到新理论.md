## 一 · 为什么不是三条旧路

从本框架看，统一有四条最小路线：

| 路线 | 形式读法 | 是否可作正向整合 |
|---|---|---|
| 直接相加 | 保留量子项和相对论项，但不新增中介 | 不可 |
| 还原到量子 | 用量子态空间吞掉事件 / 度规 / 因果 | 不可 |
| 还原到相对论 | 用时空几何吞掉态 / 幅 / 测量 | 不可 |
| 中介桥 | 新增一层，同时投影到二者 | 可作为方向 |

Lean 中这被压成：

```lean
inductive IntegrationRoute : Type
  | directAddition
  | reduceToQuantumFace
  | reduceToRelativityFace
  | mediatedBridge
```

本框架的判准是：

```lean
def isFrameworkRoute (r : IntegrationRoute) : Bool :=
  keepsQuantumFace r && keepsRelativityFace r && introducesMediator r
```

也就是说，路线必须同时满足三点：

| 判准 | 含义 |
|---|---|
| `keepsQuantumFace` | 不丢掉态空间、概率幅、测量 |
| `keepsRelativityFace` | 不丢掉事件、度规、因果结构 |
| `introducesMediator` | 不只是把旧语言并列摆放，而是新增中介层 |

于是可证：

```lean
theorem only_mediated_bridge_is_framework_route
    (r : IntegrationRoute) :
    isFrameworkRoute r = true ↔ r = .mediatedBridge
```

这句话是本文的方向定理：若统一仍要尊重“一有二面”，就不能用一面吞掉另一面，而要找能双向投影的新中介。

## 二 · 中介层应是什么

中介层不是第三套口号，而是一个带双投影的结构：

```lean
structure BridgeTerm where
  quantumProjection : QuantumSubject
  relativityProjection : RelativitySubject
```

它的物理读法是：

| 中介必须解释 | 量子投影 | 相对论投影 |
|---|---|---|
| 可能性如何组织 | 态空间 / 概率幅 | 事件尚未定域前的可显现结构 |
| 显现如何落成 | 测量 | 时空事件 |
| 传播如何受限 | 算子与相关性 | 因果结构 |
| 几何如何稳定 | 态的统计结构 | 度规 / 曲率 |

更直接地说，中介层要回答：

```
概率幅如何成为事件？
事件网络如何承载概率幅？
因果约束如何限制可测结果？
度规几何如何从稳定的显现关系中恢复？
```

如果这四个问题没有同时被回答，就还不是统一，只是局部桥接。

## 三 · 从我们的框架看，桥在“测”

本项目已有一个关键读法：量子侧偏向“虚 / 可能 / 幅”，相对论侧偏向“实 / 事件 / 因果”。二者的交界不是抽象名词，而是“测”。

测量在这里不是普通观察动作，而是一个边界：

| 边界前 | 边界动作 | 边界后 |
|---|---|---|
| 态 / 幅 / 可能性 | 测 | 事件 / 记录 / 因果位置 |
| 虚实未分 | 显现接口 | 实际可读结果 |
| 多分支结构 | 投影 / 选择 / 更新 | 一个可定位的事实 |

所以整合方向第一步应当放在：

```
测量事件接口
```

不是先问“时空是不是 Hilbert 空间”，也不是先问“波函数是不是几何”，而是先问：

> 一个测量结果何以既是 Born rule 的结果，又是时空中的事件？

这是最小交叉点。若这里无法统一，后面谈度规、曲率、量子引力都还没有落脚点。

## 四 · 中介理论需要三层投影

真正的中介不只是有两个出口，还要有三层投影：

| 投影 | 作用 | 失败时的问题 |
|---|---|---|
| 量子投影 | 回收态、幅、算子、Born rule | 只剩几何，不再是量子理论 |
| 相对论投影 | 回收事件、度规、因果结构 | 只剩态空间，不再是时空理论 |
| 测量投影 | 说明一次显现如何同时被两边读取 | 二者仍然只是并列模型 |

这三层投影对应三类问题：

| 问题 | 必要产物 |
|---|---|
| 量子如何读 | 态空间或等价的概率幅结构 |
| 时空如何读 | 事件集、因果关系、几何量 |
| 测量如何读 | 从可能性到事件记录的桥接规则 |

本轮 Lean 只证明前两类投影的形式边界：

```lean
theorem bridge_projections_same_one (b : BridgeTerm) :
    sameOne b.quantumFace b.relativityFace

theorem bridge_projections_not_current_identity (b : BridgeTerm) :
    ¬ CurrentPhysicalUnity b.quantumFace b.relativityFace
```

也就是：桥接项可以把两面接到同一个 root，但不能把它们降格为同一个旧语言对象。

## 五 · 研究路线图

从这个框架出发，整合可以分六步走。

### 1. 固定中介载体

先不要急着写“最终方程”。先确定中介载体是什么。

候选读法包括：

| 载体方向 | 框架读法 |
|---|---|
| 历史 / 路径 | 量子幅投在可能历史上，相对论事件从历史中显现 |
| 因果网络 | 节点是事件，边是因果约束，幅附着在网络或转移上 |
| 过程范畴 | 对象是边界状态，态射是过程，测量是态射的可读截面 |
| 信息几何 | 概率结构给出几何，稳定极限表现为度规 |

这些只是方向类型，不是本文的物理承诺。共同点是：它们都不是旧的量子项或旧的相对论项，而是能投影到两边的中介。

### 2. 定义双投影

中介必须有两套投影：

```
Mediator → QuantumLanguage
Mediator → RelativityLanguage
```

投到量子侧，必须能读出态、幅、测量概率。

投到相对论侧，必须能读出事件、因果、度规或其等价结构。

如果只能投一边，它只是还原论；如果两边都能投但没有兼容律，它只是翻译表。

### 3. 写兼容律

兼容律是整合的核心。

它要说明：

| 兼容对象 | 要求 |
|---|---|
| 测量概率与事件发生 | Born rule 的结果如何成为事件记录 |
| 量子相关与因果约束 | 相关性不能破坏可通信的因果结构 |
| 态演化与几何变化 | 态的变化如何对应几何或因果网络的变化 |
| 局部读数与全局结构 | 局部测量如何嵌入整体时空结构 |

本文没有给出兼容律。本文只指出：没有兼容律，中介桥还只是骨架。

### 4. 做双极限恢复

一个候选统一理论必须回收两端：

| 极限 | 必须回收 |
|---|---|
| 固定背景 / 弱引力极限 | 普通量子理论或 QFT 式结构 |
| 经典 / 大尺度极限 | 相对论时空、因果锥、度规动力学 |
| 测量记录极限 | 可重复、可共享、可定位的实验事件 |

不能回收量子侧，就不是量子统一。

不能回收相对论侧，就不是时空统一。

不能回收测量记录，就无法与经验世界相接。

### 5. 给出新预言

统一理论不能只做翻译。它最终要给出旧理论组合无法给出的差异。

差异可以来自：

| 差异来源 | 可能形态 |
|---|---|
| 高能 / 小尺度 | 旧时空连续性失效 |
| 强引力量子态 | 事件结构受量子态影响 |
| 测量边界 | 记录生成与几何结构之间出现新约束 |
| 信息极限 | 可测概率与因果结构之间有新的上界 |

本文不声称已有这些预言；这里只把它们列为闭合标准。

### 6. 回到“测”

每一条候选路线最后都要回到同一个问题：

> 这个理论如何说明一次测量结果同时是量子概率的显现，也是时空中的事件？

这就是本文所谓“从桥到新理论”的核心。
