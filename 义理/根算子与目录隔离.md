## Claim

> **算之本体不在旧目录，而在 root-native source：mask、program、projection、alias。旧目录若有用，只能作为迁移材料；它可以命名、举例、留证或删除，但不能凭目录规模证明完整。**

这回应的是一个边界问题：

```text
目录能帮助命名。
目录不能生成本体。
目录能留材料。
目录不能替代规则。
```

---

## 边界声明

本文不是说旧目录全无价值，而是说它的价值不在“数量”。

| 可保留 | 不可保留 |
|---|---|
| 旧名作为 alias | 目录大小作为 root count |
| 例句作为 corpus | 行数作为完整语义 |
| 注释作为 evidence note | grid 作为本体空间 |
| 错项作为 delete | 历史分组作为 v3 架构 |

## 完成度审计

| scope | status | anchor |
|---|---|---|

| 每个 root operator 有 R8 投影 | machineChecked | `every_operator_has_visible_projection` |
| 每个 root operator 有 loss ledger | machineChecked | `every_operator_has_loss_ledger` |
| alias 不新增行为 | machineChecked | `alias_visible`, `alias_loss` |
| legacy role 不作本体计数 | machineChecked | `no_legacy_role_is_ontology_count` |
| 每个旧目录条目逐条归档 | 未纳入本轮 | 由 quarantine audit 执行 |
| 完整自然语言语义 | 未纳入本轮 | parser / interpreter / corpus |

---

## 四类根算子

| 根算子 | 中文读法 | 形式读法 |
|---|---|---|
| `mask` | 一个 R8 变换面具 | `.path [c]` |
| `program` | 一段 root-rule 程序 | `CoreForm.eval emptyEnv e` |
| `projection` | 高维对象落到 R8 的可见影 | `.projected c loss` |
| `alias` | 旧名或别名 | 继承 target 的 visible / loss |

所以“旧词能不能留下”不是看它在旧目录里，而是看它能不能落到这四类之一。

---

## 旧目录的四种处置

| role | 处理 |
|---|---|
| `alias` | 指向已有 root operator |
| `corpus` | 进入例句与解释器测试 |
| `evidenceNote` | 留作审稿材料，不进入主 claim |
| `delete` | 移出入口，不再驱动架构 |

这使得旧目录可以被用，但不能反客为主。
