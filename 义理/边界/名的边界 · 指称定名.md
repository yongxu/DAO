## 二 · 为什么名不是知

可名只是指称成功。可知还要求明、辨、述、贯、存，使对象进入可描述之理。

```text
“某星”可以被暂名，但其结构未必已知。
```

所以：

```text
可名 ≠ 可知
```

名可以给知提供入口，但不能替代知。

---

## 三 · 为什么名不是证

证明需要命题、系统、推导、检查器、证书。名只需要指称流程。

```text
给命题取名 ≠ 证明命题
```

所以：

```text
可名 ≠ 可证
```

证书可以有名字，名字不是证书本身。

## 四 · Handoff

此前 `KnowableBoundary.lean` 只枚举可名 / 不可名，声明本文件不处理。`KeBoundary.lean` 也明确不处理可名 / 不可名。

现在由本文承接：

```lean
theorem name_deferred_boundaries_handled_in_ming :
    handledHere .nameable = false ∧ handledInKeBoundary .nameable = false
    ∧ handledInMingBoundary .nameable = true
    ∧ handledHere .unnameable = false ∧ handledInKeBoundary .unnameable = false
    ∧ handledInMingBoundary .unnameable = true
```

这表示：

```text
可名 / 不可名不是“可知”文件的职责；
也不是“感 / 行 / 证”文件的职责；
它由“名”的边界承接。
```

## 五 · 主定理

完整名进入可名：

```lean
theorem complete_ming_maps_to_nameable (m : MingSkeleton) :
    CompleteMing m → Nameable (mingAsNamingRegion m)
```

缺失轴落入不可名：

```lean
theorem incomplete_ming_maps_to_unnameable
    (m : MingSkeleton)
    (h : ¬ CompleteMing m) :
    mingAsNamingRegion m = .unnameable
```
