## 〇 · 边界声明

本文处理“可行”的流程细化，但不展开完整行动理论。

| 命题 | 本文处理方式 | 不声称 |
|---|---|---|
| 什么算可行 | 五轴完整时进入 `Actionable` | 不声称行动一定成功或值得做 |
| 不可行如何判定 | 任一轴缺失则落到 `unactionable` | 不判定现实世界的全部障碍 |
| 行与知关系 | 分层处理 | 不把“知道怎么做”等同于“能做” |
| 行与证关系 | 分层处理 | 不把“证明可行”等同于“已经可行” |
| 行与感关系 | 分层处理 | 不把“感觉能做”等同于“可行” |

## 〇一 · 完成度审计

本轮“行”闭合到五条轴：

| 轴 | 内侧读法 | 外侧读法 | Lean 状态 | theorem |
|---|---|---|---|---|
| 主体 `agent` | 有行动承载者 | 无承载者 | 已证 typed skeleton | `xing_axes_pairwise_distinct` |
| 能力 `capacity` | 有执行能力 | 能力缺失 | 已证 typed skeleton | `complete_xing_maps_to_actionable` |
| 境况 `context` | 情境允许行动 | 情境不允许 | 已证 typed skeleton | `complete_xing_maps_to_actionable` |
| 操作 `operation` | 有可执行步骤 | 没有操作路径 | 已证 typed skeleton | `complete_xing_maps_to_actionable` |
| 效果 `effect` | 有可验效果接口 | 无效果落点 | 已证 typed skeleton | `complete_xing_maps_to_actionable` |

一句话范围：**主体、能力、境况、操作、效果五轴已闭合；价值判断、伦理正当性、长期后果、完整因果动力学不在本文。**

## 一 · 主映射

Lean 中：

```lean
structure XingSkeleton where
  agent : BoundarySide
  capacity : BoundarySide
  context : BoundarySide
  operation : BoundarySide
  effect : BoundarySide
```

完整行：

```lean
def CompleteXing (x : XingSkeleton) : Prop :=
  x.agent = .inside
    ∧ x.capacity = .inside
    ∧ x.context = .inside
    ∧ x.operation = .inside
    ∧ x.effect = .inside
```

映射规则：

| 流程状态 | 映射到 |
|---|---|
| 五轴都在内侧 | `ActionabilityRegion.actionable` |
| 任一轴缺失 | `ActionabilityRegion.unactionable` |

这把“可行”从口头判断压成接口判断：

```text
谁能做？凭什么能做？在什么境况中做？怎么做？做到哪里算落地？
```

---

## 二 · 为什么行不是知

知道一个行动方案，不等于具备主体、能力、境况、操作和效果。

```text
知道登月方案 ≠ 当前主体可登月
```

知的界限是可描述之理；行的界限是实践接口。二者可以接续，但不能同一。

---

## 三 · 为什么行不是证

证明某行动在模型中可行，不等于现实行动已经可行。

```text
proof interface 内可证
```

与：

```text
行动接口内可执行
```

是两条边界。证可以给行担保，但不能替代主体、能力、境况、操作和效果。

---

## 四 · 主定理

完整行进入可行：

```lean
theorem complete_xing_maps_to_actionable (x : XingSkeleton) :
    CompleteXing x → Actionable (xingAsActionability x)
```

缺失轴落入不可行：

```lean
theorem incomplete_xing_maps_to_unactionable
    (x : XingSkeleton)
    (h : ¬ CompleteXing x) :
    xingAsActionability x = .unactionable
```
