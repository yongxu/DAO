## 一 · 统一是什么

统一不是：

```
一条命题
一个 bit
一个模型
一段文字
一个物理公式
```

这些都属于信息描述。信息描述的特点是：

| 条件 | 含义 |
|---|---|
| 有差异 | 至少能区分此/彼、真/假、0/1、词/词 |
| 有载体 | token、公式、图像、测量记录 |
| 有边界 | 说了这一句，就没有同时说全部可能句 |
| 可传递 | 可以编码、复制、引用、解析 |

统一不是这些对象之一。

更准确地说：

> 统一是信息描述能够被归到同一场域中比较的条件。

也就是说，统一不是“被描述的信息”，而是“描述得以互相对齐的边界”。

Lean 中压成两个类型：

```lean
inductive DescriptionContent : Type
  | finiteContent
  | boundaryPointer

def unityContent : DescriptionContent := .boundaryPointer
```

普通信息描述只落到 `finiteContent`：

```lean
def contentOfInfo : InformationDescription → DescriptionContent
  | .bit _ => .finiteContent
  | .finiteText _ => .finiteContent
```

因此有：

```lean
theorem information_description_never_is_unity (d : InformationDescription) :
    contentOfInfo d ≠ unityContent
```

这就是“统一不可作为信息描述”的最小证明。

---

## 二 · 为什么统一不能是信息描述

信息描述依赖分割。

最小的信息是 bit：

```
0 / 1
```

但统一若被当成一个 bit，就已经被放进了差异系统：

```
unity / not-unity
```

这会把统一变成一个局部对象。局部对象可以被比较、否定、编码、替换，但它不再是统一本身。

所以统一可以被命名、指示、建模、证明其边界；但不能作为 ordinary information object 被完整给出。

Lean 中对应：

```lean
inductive UnityRole : Type
  | informationObject
  | sharedBoundaryCondition

def unity : UnityRole := .sharedBoundaryCondition
```

证明：

```lean
theorem unity_not_information_object :
    unity ≠ UnityRole.informationObject

theorem unity_is_shared_boundary_condition :
    unity = UnityRole.sharedBoundaryCondition
```

这不是语言游戏，而是边界分层：

| 层 | 能做什么 | 不能做什么 |
|---|---|---|
| 信息层 | 说出一个内容 | 说出让所有内容可归一的条件本身 |
| 理层 | 给出形式证明 | 把元边界变成对象层全部内容 |
| 文层 | 命名、显象、指路 | 用有限字句穷尽无限边界 |
| 统一层 | 使描述可归、可比、可互译 | 作为一条普通消息被消费 |

## 三 · 理的极限是什么

理不是空想。理是形式、证明、条贯、可检查结构。

但理的极限在于：理一旦成为表达，就必须有限化。

在 Lean 中，本文把理表达压成：

```lean
structure LiExpression where
  tokens : Nat
```

这不是说理只有 token；而是说任何已经进入可检查接口的理，都有有限承载。

有限承载必有外部深度：

```lean
def HasOutsideDepth (e : FiniteExpression) : Prop :=
  ∃ n : Nat, e.tokenCount < n
```

证明：

```lean
theorem li_expression_has_outside_depth (l : LiExpression) :
    liHasOutsideDepth l
```

其含义是：

| 理能做到 | 理的极限 |
|---|---|
| 明确公理、定义、推理、证明 | 不能把所有元条件都压成对象层内容 |
| 给出边界 theorem | theorem 仍在某个表达层内 |
| 审计一致性 | 不能从有限内部穷尽全部外部深度 |

这与既有 `DaoLi.lean` 的方向一致：

| theorem | 含义 |
|---|---|
| `dao_proves_about_li` | 道层能证明理层的元命题 |
| `li_cannot_encode_dao` | 理层有限程序不能编码道层无界塔 |
| `dao_li_bifurcation` | 道可论理，理不穷道 |

所以理的极限不是失败，而是理的诚实边界：理能证明自己到哪里，不能伪装自己覆盖哪里。

---

## 四 · 文字表达的极限是什么

文字不是低于理的东西。文字让边界显形，让结构可传，让经验可共。

但文字一旦成为表达，也必须有限化：

```lean
structure WenExpression where
  tokens : Nat
```

每一篇文章、每一句话、每一个命名，都是有限表达。有限表达同样有外部深度：

```lean
theorem wen_expression_has_outside_depth (w : WenExpression) :
    wenHasOutsideDepth w
```

文字的极限是：

| 文能做到 | 文的极限 |
|---|---|
| 命名 | 名不是所名之全体 |
| 描写 | 描写不是被描写者本身 |
| 指示边界 | 指示不是吞没边界 |
| 让理可读 | 可读不等于穷尽 |

因此，“不可说”不是“不能说”。更精确是：

> 能说，但所说总是 finite pointer；不能把 pointer 当成 unified whole。

## 五 · 为什么理限与文限一致

理与文看似不同：

| 理 | 文 |
|---|---|
| 定义、定理、证明 | 字、句、篇、象 |
| 追求无歧义 | 容纳多义与指示 |
| 适合检查 | 适合传达 |

但在表达边界上，它们同构：

```
LiExpression ─┐
              ├─► FiniteExpression ─► sharedLimit
WenExpression ┘
```

Lean 中：

```lean
def LiExpression.toFinite (e : LiExpression) : FiniteExpression :=
  ⟨e.tokens⟩

def WenExpression.toFinite (e : WenExpression) : FiniteExpression :=
  ⟨e.tokens⟩
```

二者的边界都由同一个函数给出：

```lean
def finiteExpressionBoundary (_e : FiniteExpression) : BoundaryKind :=
  .sharedLimit
```

所以可证明：

```lean
theorem li_wen_boundaries_coincide (l : LiExpression) (w : WenExpression) :
    liLimit l = wenLimit w
```

这就是“理的极限”和“文字表达的极限”一致。

一致不表示理和文没有差别；一致表示它们在“有限表达不能穷尽统一”这一点上共享同一边界。

## 六 · 统一、理、文的三层关系

可以把结构写成：

```
统一
  │
  ├─ 理：以定义和证明指向边界
  │
  └─ 文：以文字和象指向边界
```

理与文都不是统一本身。

但理与文都可以成为统一的指针：

| 项 | 作为内容 | 作为指针 |
|---|---|---|
| 理 | theorem / model / proof | 指向形式边界 |
| 文 | phrase / image / narrative | 指向表达边界 |
| 统一 | 不作为普通内容 | 作为 shared boundary condition |

所以统一不是“理 + 文”的总和。

统一是使理与文能在同一边界处相遇的条件。

## 七 · 对“证明”的重新定位

本文所谓证明，不是证明统一本身被 Lean 完全捕获。

Lean 证明的是：

| 可证明对象 | Lean theorem |
|---|---|
| 普通信息描述不能给出统一内容 | `information_description_never_is_unity` |
| 统一的角色是 shared boundary condition | `unity_is_shared_boundary_condition` |
| 理表达有外部深度 | `li_expression_has_outside_depth` |
| 文表达有外部深度 | `wen_expression_has_outside_depth` |
| 理限与文限一致 | `li_wen_boundaries_coincide` |
| 若理文 tokenCount 相同，则外部深度谓词等价 | `li_wen_outside_depth_equiv_of_same_count` |

这组证明的意义是：它把“统一不可作为信息描述”从一句玄言，压成了一个可审计的类型边界。

## 八 · 主定理

综合 theorem：

```lean
theorem unity_boundary_summary :
    (∀ d : InformationDescription, contentOfInfo d ≠ unityContent)
    ∧ unity = UnityRole.sharedBoundaryCondition
    ∧ (∀ l : LiExpression, ∀ w : WenExpression, liLimit l = wenLimit w)
    ∧ (∀ l : LiExpression, liHasOutsideDepth l)
    ∧ (∀ w : WenExpression, wenHasOutsideDepth w)
```

读法：

1. 统一不是 ordinary information content；
2. 统一是 shared boundary condition；
3. 理限与文限一致；
4. 理与文都作为 finite expression 有其外部深度。

---

## 九 · 结论

统一是什么？

```
不是一条信息；
不是一个对象；
不是一个可消费的结论；
而是信息、对象、结论得以互相对齐的共同边界。
```

理的极限是什么？

```
理能证明边界；
但不能把边界完整装入对象层。
```

文字表达的极限是什么？

```
文能命名边界；
但不能把边界完整变成有限字句。
```

为什么二者一致？

```
因为理与文一旦进入表达，都必须有限化；
有限表达必有外部深度；
这个外部深度就是二者共同抵达的 sharedLimit。
```

所以本文的最终判断是：

> 统一不可作为信息描述；统一是理与文共同抵达、共同指向、但不能共同吞没的边界。
