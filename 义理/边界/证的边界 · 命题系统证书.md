## 〇 · 边界声明

本文只做证明边界的 typed skeleton。

| 命题 | 本文处理方式 | 不声称 |
|---|---|---|
| 什么算可证 | 五轴完整时进入 `ProofRegion.provable` | 不展开完整证明论 |
| 不可证如何处理 | 任一轴缺失落到 `unprovable` | 不证明哥德尔、不独立性判定 |
| 证与真关系 | 不处理 | 不把可证等同真 |
| 证与知关系 | 证的边界可说 | 不把可知压成可证 |
| 证与推关系 | 证比一般推更强 | 不把所有推理都叫证明 |

## 一 · 主映射

Lean 中：

```lean
structure ZhengSkeleton where
  proposition : BoundarySide
  system : BoundarySide
  derivation : BoundarySide
  checker : BoundarySide
  certificate : BoundarySide
```

完整证：

```lean
def CompleteZheng (z : ZhengSkeleton) : Prop :=
  z.proposition = .inside
    ∧ z.system = .inside
    ∧ z.derivation = .inside
    ∧ z.checker = .inside
    ∧ z.certificate = .inside
```

映射规则：

| 流程状态 | 映射到 |
|---|---|
| 五轴都在内侧 | `ProofRegion.provable` |
| 任一轴缺失 | `ProofRegion.unprovable` |

## 二 · 为什么证不是一般推

推只要求前提、规则、推导、结论、担保形成 inference claim。证还要求命题被置入系统，并能交给检查器生成或核验证书。

所以：

```text
推得出结论 ≠ 已有可复核证明证书
```

推可以是证的一部分，但证不是任意推论。

---

## 三 · 为什么证不是知

可知的边界是可描述之理；可证的边界是当前 proof interface。

某内容可以可知但未可证：

```text
可说、可归理、可审计，但尚未在指定系统中给出证书
```

某证明对象也必须可说，因为 proof boundary 本身仍由语言表达：

```lean
theorem zheng_boundary_is_sayable_skeleton (z : ZhengSkeleton) :
    Sayable (proofRegionAsKnowable (zhengAsProofRegion z))
```

这只是说“证的边界可被说出”，不是说“凡可说皆可证”。

## 四 · 主定理

完整证进入可证：

```lean
theorem complete_zheng_maps_to_provable (z : ZhengSkeleton) :
    CompleteZheng z → Provable (zhengAsProofRegion z)
```

缺失轴落入不可证：

```lean
theorem incomplete_zheng_maps_to_unprovable
    (z : ZhengSkeleton)
    (h : ¬ CompleteZheng z) :
    zhengAsProofRegion z = .unprovable
```
