## 〇一 · 完成度审计

本轮“知”闭合到五条轴：

| 轴 | 内侧读法 | 外侧读法 | Lean 状态 | theorem |
|---|---|---|---|---|
| 明 `access` | 对象进入认知接口 | 对象未进入接口 | 已证 typed skeleton | `zhi_axes_pairwise_distinct` |
| 辨 `distinction` | 能与其他对象区分 | 不能稳定区分 | 已证 typed skeleton | `zhi_axes_pairwise_distinct` |
| 述 `articulation` | 能被表述为可审计内容 | 不能表述为理性内容 | 已证 typed skeleton | `complete_zhi_maps_to_knowable` |
| 贯 `coherence` | 能接入已有理路 | 与当前理路不成贯通 | 已证 typed skeleton | `complete_zhi_maps_to_knowable` |
| 存 `retention` | 能稳定保存、复用、复核 | 不能稳定保留 | 已证 typed skeleton | `complete_zhi_maps_to_knowable` |

一句话范围：**明、辨、述、贯、存五轴已闭合；经验真实性、意识状态、证明论强度、全知问题不在本文。**

## 一 · 主映射

Lean 中：

```lean
structure ZhiSkeleton where
  access : BoundarySide
  distinction : BoundarySide
  articulation : BoundarySide
  coherence : BoundarySide
  retention : BoundarySide
```

完整知：

```lean
def CompleteZhi (z : ZhiSkeleton) : Prop :=
  z.access = .inside
    ∧ z.distinction = .inside
    ∧ z.articulation = .inside
    ∧ z.coherence = .inside
    ∧ z.retention = .inside
```

映射规则：

| 流程状态 | 映射到 |
|---|---|
| 五轴都在内侧 | `KnowableRegion.withinDescribableLi` |
| 任一轴缺失 | `KnowableRegion.beyondDescribableLi` |

所以“知”的形式版不是心理瞬间，而是一个接口判断：

```text
能明 → 能辨 → 能述 → 能贯 → 能存
```

五者齐备，才落入“可描述之理”的知界。

---

## 二 · 为什么知不是名

名只要求指称、符号、约定、论域、消歧成立。知还要求该内容能被贯通和保存为可审计之理。

一个名字可以先于知识出现：

```text
“X”这个符号有约定，但 X 的结构尚未被明辨述贯存。
```

所以：

```text
可名 ≠ 可知
```

名是入口，知是可描述之理的完成接口。

---

## 三 · 为什么知不是证

可证比可知更窄。一个内容可能已经能被说明、归理、保存，但还没有进入某个形式 proof interface。

```text
可知：能作为理被描述
可证：能在当前系统中给出证明证书
```

所以：

```text
可知 ≠ 可证
```

本文只把“知”对齐到 `Knowable`，不把它进一步压成 `Provable`。

## 四 · 主定理

完整知进入可知：

```lean
theorem complete_zhi_maps_to_knowable (z : ZhiSkeleton) :
    CompleteZhi z → Knowable (zhiAsKnowable z)
```

缺失轴落在可描述之理之外：

```lean
theorem incomplete_zhi_maps_beyond_describable_li
    (z : ZhiSkeleton)
    (h : ¬ CompleteZhi z) :
    zhiAsKnowable z = .beyondDescribableLi
```
