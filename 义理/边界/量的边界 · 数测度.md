## 〇 · 边界声明

本文证明的是结构边界，不是经验科学定理。

| 命题 | 本文处理方式 | 不声称 |
|---|---|---|
| 量有五条不同轴 | Lean 证明构造子不同 | 不声称自然语言中只有五种“量” |
| 每条轴有内外两侧 | typed skeleton 穷尽互斥 | 不解决每个具体对象落在哪侧 |
| 可测由量承接 | Lean handoff 从 `DeferredKeBoundary` 到 `LiangBoundary` | 不把可测并回可知、可感、可证 |
| Kolmogorov 概率归一 | 接入已有 `ProbabilityMeasure` 证明 | 不重建完整测度论 |
| Born rule | 接入 computational-basis 二值版本 | 不证明一般 POVM、退相干或测量问题 |
| 测量输出 | 接入实 / 虚 / 八卦态三类输出 skeleton | 不声称物理实验只有三种实际结果 |

这里的标准很窄：**只证明接口分层和边界闭合**。一旦涉及具体物理模型、实验仪器、Lebesgue 构造、量子测量公设，就必须进入相应理论文件，而不是由“量”的总边界直接替代。

## 一 · 为什么“量”不能只作一个词

“量”至少有五种常见读法：

| 读法 | 问题 | 例子 |
|---|---|---|
| 数量 | 有多少 / 值是多少 | 3、1/2、64 |
| 度量 | 能否比较大小、距离、尺度 | 长度、范数、误差、方差 |
| 测度 | 哪些事件可测，测度如何分配 | σ-代数、概率测度、归一 |
| 测量 | 如何从系统得到可读结果 | 观测、collapse outcome、实验记录 |
| 量词 | 断言覆盖哪个论域 | ∀、∃、“所有”、“存在” |

这五者经常相互依赖，但不能互相代替。

一个数值可以没有测量来源，例如形式系统里的自然数。
一个测量可以产生非数值结果，例如分类标签。
一个度量可以不归一成概率。
一个概率测度需要事件域与可加律，不只是把一个数字放在 `[0,1]`。
一个量词若无论域，就不是合法的全称或存在断言。

Lean 中先把五轴分开：

```lean
inductive LiangAxis : Type
  | quantity
  | metric
  | measureTheory
  | measurement
  | quantifier
```

并证明它们两两不同：

```lean
theorem liang_axes_pairwise_distinct :
    LiangAxis.quantity ≠ LiangAxis.metric
    ∧ LiangAxis.quantity ≠ LiangAxis.measureTheory
    ∧ LiangAxis.quantity ≠ LiangAxis.measurement
    ∧ LiangAxis.quantity ≠ LiangAxis.quantifier
    ∧ LiangAxis.metric ≠ LiangAxis.measureTheory
    ∧ LiangAxis.metric ≠ LiangAxis.measurement
    ∧ LiangAxis.metric ≠ LiangAxis.quantifier
    ∧ LiangAxis.measureTheory ≠ LiangAxis.measurement
    ∧ LiangAxis.measureTheory ≠ LiangAxis.quantifier
    ∧ LiangAxis.measurement ≠ LiangAxis.quantifier
```

这一步的意义是：之后所有“量”的讨论，都必须先问它落在哪一条轴上。

## 二 · 数量：值，不是测量本身

数量边界问：

```text
某对象是否已经成为可作为值处理的 quantity？
```

Lean 中：

```lean
inductive QuantityRegion : Type
  | quantitative
  | nonquantitative
```

主证明：

```lean
theorem quantity_boundary_complete :
    (∀ r : QuantityRegion, Quantitative r ∨ Nonquantitative r)
    ∧ (∀ r : QuantityRegion, Quantitative r → ¬ Nonquantitative r)
    ∧ (∀ r : QuantityRegion, Nonquantitative r → ¬ Quantitative r)
```

读法：

| 结论 | 含义 |
|---|---|
| 穷尽 | 在数量接口中，只问是否已成 quantity |
| 互斥 | 同一对象在同一接口下不能既已数值化又未数值化 |

数量是值层，不自动包含来源。一个值可以由定义给出、由计算给出、由测量给出，也可以只是变量占位。

因此：

```text
有数量 ≠ 已测量
```

## 三 · 度量：尺度，不是概率

度量边界问：

```text
某对象是否进入尺度 / 距离 / 大小比较接口？
```

Lean 中：

```lean
inductive MetricRegion : Type
  | metricable
  | unmetricable
```

主证明：

```lean
theorem metric_boundary_complete :
    (∀ r : MetricRegion, Metricable r ∨ Unmetricable r)
    ∧ (∀ r : MetricRegion, Metricable r → ¬ Unmetricable r)
    ∧ (∀ r : MetricRegion, Unmetricable r → ¬ Metricable r)
```

度量让比较成立：

| 度量对象 | 典型形式 |
|---|---|
| 距离 | `d(x, y)` |
| 大小 | norm / magnitude |
| 偏差 | `|x - μ|` |
| 分散 | variance / spread |

但度量不是概率。方差是度量式量；概率是测度式量。二者会在统计中耦合，但语义位置不同。

## 四 · 测度：事件域、可加、归一

测度边界问：

```text
某事件是否在当前 σ-代数 / measurable interface 内？
```

Lean 中：

```lean
inductive MeasureRegion : Type
  | sigmaMeasurable
  | sigmaUnmeasurable
```

主证明：

```lean
theorem measure_theoretic_boundary_complete :
    (∀ r : MeasureRegion, SigmaMeasurable r ∨ SigmaUnmeasurable r)
    ∧ (∀ r : MeasureRegion, SigmaMeasurable r → ¬ SigmaUnmeasurable r)
    ∧ (∀ r : MeasureRegion, SigmaUnmeasurable r → ¬ SigmaMeasurable r)
```

这接上《统计 · 从元到测》的核心：

```text
事：event
域：sample space / σ-algebra
度：measure
率：probability
合：additivity
一：normalization
因：conditioning
```

关键不是“有一个概率数字”，而是：

```text
事件域 + 可测性 + 可加性 + 归一
```

Lean 中由 Mathlib 的 `ProbabilityMeasure` 给出归一锚点：

```lean
theorem probability_measure_normalizes
    (α : Type*) [MeasurableSpace α] (P : ProbabilityMeasure α) :
    (P : Measure α) Set.univ = 1
```

这说明概率测度不是任意 `[0,1]` 函数。它必须以 measurable space 为底，并满足总量为一。

## 五 · 测量：接口输出，不等于测度可测

测量边界问：

```text
某对象是否能经观测 / 测量接口落成可读结果？
```

Lean 中：

```lean
inductive MeasurementRegion : Type
  | measurableByInterface
  | unmeasurableByInterface
```

主证明：

```lean
theorem measurement_boundary_complete :
    (∀ r : MeasurementRegion, MeasurableByInterface r ∨ UnmeasurableByInterface r)
    ∧ (∀ r : MeasurementRegion, MeasurableByInterface r → ¬ UnmeasurableByInterface r)
    ∧ (∀ r : MeasurementRegion, UnmeasurableByInterface r → ¬ MeasurableByInterface r)
```

这里必须区分两种“可测”：

| 名称 | 所属轴 | 问题 |
|---|---|---|
| σ-可测 | 测度 | 事件是否属于可测事件族 |
| 接口可测 | 测量 | 系统是否能输出可读结果 |

量子文档中的 collapse outcome 属于第二种。它不是说某个事件属于 σ-代数，而是说一次 typed measurement 的输出已经进入可读接口。

本节只判定是否落入 measurement 内侧；完整测量流程的观、接、法、校、读、记六轴由 [测的边界 · 观验读](%E6%B5%8B%E7%9A%84%E8%BE%B9%E7%95%8C%20%C2%B7%20%E8%A7%82%E9%AA%8C%E8%AF%BB.md) 细化。

Lean 接口：

```lean
def collapseOutcomeMeasurementRegion (_o : CollapseOutcome) : MeasurementRegion :=
  .measurableByInterface

theorem every_typed_measurement_lands_in_liang_measurement_region
    (m : Measurement) (s : PreMeasurement) :
    MeasurableByInterface (collapseOutcomeMeasurementRegion (m s))
```

并接入此前的三类输出限制：

```lean
theorem collapse_output_has_no_fourth_liang_class (o : CollapseOutcome) :
    withinMeasurementLimit o
```

读法：在那个量子时空 skeleton 里，输出只落为实、虚、或八卦态；本文只说明这类输出一旦形成，就在“量”的测量接口内侧。

## 六 · Born rule：从概率幅到测量权重

Born rule 在本文只接入 one-qubit computational basis 版本：

```lean
theorem born_rule_gives_normalized_measurement_weights
    (ψ : Qubit) (hψ : computationalBasisNormalized ψ) :
    0 ≤ bornProb0 ψ ∧ 0 ≤ bornProb1 ψ
      ∧ bornProb0 ψ + bornProb1 ψ = 1
```

这给出三件事：

| 条件 | 含义 |
|---|---|
| `0 ≤ bornProb0 ψ` | `|0⟩` 分支权重非负 |
| `0 ≤ bornProb1 ψ` | `|1⟩` 分支权重非负 |
| `bornProb0 ψ + bornProb1 ψ = 1` | 归一化 qubit 的二值测量权重和为一 |

这比“从虚到实之概率投影”更形式化，但仍是窄版本：

```text
computational basis / one qubit / normalized state
```

未纳入本轮：

| 项 | 原因 |
|---|---|
| 一般 Hilbert 空间测量 | 需要更大的 Mathlib/物理结构 |
| POVM / projection-valued measures | 需要算子谱与测量公设 |
| 退相干 | 需要动力学与环境模型 |
| 测量问题 | 不是 typed skeleton 可直接解决的问题 |

## 七 · 量词：没有论域，就没有全称

量词边界问：

```text
∀ / ∃ 覆盖哪个 domain？
```

Lean 中：

```lean
inductive QuantifierRegion : Type
  | scoped
  | unscoped
```

主证明：

```lean
theorem quantifier_boundary_complete :
    (∀ r : QuantifierRegion, Scoped r ∨ Unscoped r)
    ∧ (∀ r : QuantifierRegion, Scoped r → ¬ Unscoped r)
    ∧ (∀ r : QuantifierRegion, Unscoped r → ¬ Scoped r)
```

这条轴很容易被忽略，但它决定表达的合法性：

| 表达 | 问题 |
|---|---|
| “所有对象都有 P” | 所有对象是哪一类？ |
| “存在一个 x” | x 属于哪个类型或集合？ |
| “任意测量都……” | 测量接口是哪一个？ |

没有论域的量词不是更强，而是未定。它不该被当成全称断言。

## 八 · 可测 / 不可测的承接

此前《可知边界 · 语言与理》和《可的边界 · 感行证》都把可测 / 不可测留出。

现在 Lean 里把交接写清楚：

```lean
def handledInLiangBoundary : DeferredKeBoundary → Bool
  | .measurable
  | .unmeasurable => true
  | .feelable
  | .unfeelable
  | .actionable
  | .unactionable
  | .provable
  | .unprovable => false

theorem measurable_deferred_boundaries_handled_in_liang :
    handledHere .measurable = false ∧ handledInKeBoundary .measurable = false
    ∧ handledInLiangBoundary .measurable = true
    ∧ handledHere .unmeasurable = false ∧ handledInKeBoundary .unmeasurable = false
    ∧ handledInLiangBoundary .unmeasurable = true
```

读法：

```text
KnowableBoundary 不处理可测；
KeBoundary 也不处理可测；
LiangBoundary 承接可测 / 不可测。
```

这避免了一个常见混淆：

```text
可测 ≠ 可知
可测 ≠ 可感
可测 ≠ 可证
```

可测是“量”的边界，既可能表现为测度论可测，也可能表现为测量接口可读。它和可知、可感、可证有关，但不是它们的别名。

## 九 · 主定理

本文的总入口：

```lean
theorem liang_boundary_summary :
    (LiangAxis.quantity ≠ LiangAxis.metric
      ∧ LiangAxis.quantity ≠ LiangAxis.measureTheory
      ∧ LiangAxis.quantity ≠ LiangAxis.measurement
      ∧ LiangAxis.quantity ≠ LiangAxis.quantifier
      ∧ LiangAxis.metric ≠ LiangAxis.measureTheory
      ∧ LiangAxis.metric ≠ LiangAxis.measurement
      ∧ LiangAxis.metric ≠ LiangAxis.quantifier
      ∧ LiangAxis.measureTheory ≠ LiangAxis.measurement
      ∧ LiangAxis.measureTheory ≠ LiangAxis.quantifier
      ∧ LiangAxis.measurement ≠ LiangAxis.quantifier)
    ∧ ...
    ∧ (∀ b : DeferredKeBoundary,
        handledInLiangBoundary b = true ↔ b = .measurable ∨ b = .unmeasurable)
    ∧ ...
    ∧ (∀ (α : Type*) [MeasurableSpace α] (P : ProbabilityMeasure α),
        (P : Measure α) Set.univ = 1)
    ∧ (∀ ψ : Qubit, computationalBasisNormalized ψ →
        0 ≤ bornProb0 ψ ∧ 0 ≤ bornProb1 ψ
          ∧ bornProb0 ψ + bornProb1 ψ = 1)
    ∧ (∀ (m : Measurement) (s : PreMeasurement),
        MeasurableByInterface (collapseOutcomeMeasurementRegion (m s)))
    ∧ (∀ o : CollapseOutcome, withinMeasurementLimit o)
```

总读法：

1. 五条“量”轴不同；
2. 五条轴各自内外穷尽且互斥；
3. 可测 / 不可测由“量”承接；
4. 概率测度归一；
5. computational-basis Born rule 给非负归一测量权重；
6. 量子时空 skeleton 的测量输出进入测量接口且无第四类。

## 十 · 结论

“量”的闭合不是把一切变成数字，而是把不同量化接口放回各自位置：

```text
量
├─ 数量：值是否成立
├─ 度量：尺度 / 距离 / 比较是否成立
├─ 测度：事件是否进入 σ-代数与可加归一结构
├─ 测量：系统是否经接口落成可读结果
└─ 量词：断言是否有明确论域
```

因此“可测 / 不可测”不能直接塞回“可知”或“可感”。它现在有自己的承接层：量。
