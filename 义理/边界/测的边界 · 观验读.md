## 〇 · 边界声明

本文证明的是测量流程的结构边界。

| 命题 | 本文处理方式 | 不声称 |
|---|---|---|
| 测有六条流程轴 | Lean 证明构造子不同 | 不声称所有实际测量只需六个工程步骤 |
| 每条轴内外穷尽互斥 | typed skeleton | 不判断具体实验是否达标 |
| 完整测量落到“量”的 measurement 内侧 | `CompleteCe → MeasurableByInterface` | 不把所有可量化对象都说成已测量 |
| 缺项流程落到 measurement 外侧 | `¬ CompleteCe → unmeasurableByInterface` | 不讨论缺项后是否可补救 |
| 测量记录不是推断命题 | `measurementRecord ≠ inferenceClaim` | 不否认测量记录可被用于推断 |

这条边界尤其重要：

```text
测 ≠ 量
测 ≠ 推
测 ≠ 感
```

“量”问某对象是否进入数量、度量、测度、测量、量词接口。
“测”问一个观测流程是否足以产生可读、可复核的记录。
“推”问如何从记录推出模型、参数、假设或原因。
“感”问是否进入感受 / 现象接口。

## 一 · 为什么“测”不是“量”

《量的边界 · 数测度》已经把 measurement 作为“量”的一条轴：

```lean
inductive LiangAxis : Type
  | quantity
  | metric
  | measureTheory
  | measurement
  | quantifier
```

但这只是把“测量”放到正确的量化位置。它没有展开测量流程。

本文展开的是：

```text
对象 ──观──► 接口 ──法──► 校准 ──读──► 读数 ──记──► 记录
```

所以：

| 层 | 问题 |
|---|---|
| 量 | 这个对象是否进入某种量化接口？ |
| 测 | 这个结果是否由一套可说明的观测流程产生？ |

一个数值可以不是测得的：

```text
π、2 + 2、形式变量 x
```

一个测量也可以先不产出数值：

```text
阳性 / 阴性、通过 / 未通过、实 / 虚 / 八卦态
```

因此，测是量的一条重要入口，但不能和量整体混同。

## 二 · 六条“测”轴

Lean 中先把六轴分开：

```lean
inductive CeAxis : Type
  | observability
  | interface
  | protocol
  | calibration
  | readout
  | record
```

并证明它们两两不同：

```lean
theorem ce_axes_pairwise_distinct :
    CeAxis.observability ≠ CeAxis.interface
    ∧ CeAxis.observability ≠ CeAxis.protocol
    ∧ CeAxis.observability ≠ CeAxis.calibration
    ∧ CeAxis.observability ≠ CeAxis.readout
    ∧ CeAxis.observability ≠ CeAxis.record
    ∧ CeAxis.interface ≠ CeAxis.protocol
    ∧ CeAxis.interface ≠ CeAxis.calibration
    ∧ CeAxis.interface ≠ CeAxis.readout
    ∧ CeAxis.interface ≠ CeAxis.record
    ∧ CeAxis.protocol ≠ CeAxis.calibration
    ∧ CeAxis.protocol ≠ CeAxis.readout
    ∧ CeAxis.protocol ≠ CeAxis.record
    ∧ CeAxis.calibration ≠ CeAxis.readout
    ∧ CeAxis.calibration ≠ CeAxis.record
    ∧ CeAxis.readout ≠ CeAxis.record
```

读法：

| 轴 | 问题 | 常见误读 |
|---|---|---|
| 可观 | 对象是否进入观测窗口 | 把不可观当作已测失败 |
| 可接 | 是否有接口 / 仪器接入 | 把想观察当作已有接口 |
| 有法 | 是否有协议 / 问法 | 把随手读取当作测量 |
| 已校 | 是否有参照 / 零点 / 尺度 | 把任意数值当作可靠读数 |
| 可读 | 是否产生输出 | 把仪器存在当作已有结果 |
| 可记 | 是否进入可复核记录 | 把瞬间感受当作测量事实 |

## 三 · 可观：对象进入窗口

可观边界问：

```text
被测对象是否能进入观测窗口？
```

Lean 中：

```lean
inductive ObservabilityRegion : Type
  | observable
  | unobservable
```

主证明：

```lean
theorem observability_boundary_complete :
    (∀ r : ObservabilityRegion, Observable r ∨ Unobservable r)
    ∧ (∀ r : ObservabilityRegion, Observable r → ¬ Unobservable r)
    ∧ (∀ r : ObservabilityRegion, Unobservable r → ¬ Observable r)
```

不可观不等于不存在；它只说明在当前观测窗口内没有入口。

## 四 · 可接：接口不是愿望

接口边界问：

```text
是否已有观测接口 / 仪器 / 探针接上对象？
```

Lean 中：

```lean
inductive InterfaceRegion : Type
  | interfaced
  | uninterfaced
```

主证明：

```lean
theorem interface_boundary_complete :
    (∀ r : InterfaceRegion, Interfaced r ∨ Uninterfaced r)
    ∧ (∀ r : InterfaceRegion, Interfaced r → ¬ Uninterfaced r)
    ∧ (∀ r : InterfaceRegion, Uninterfaced r → ¬ Interfaced r)
```

这条轴把“有对象”与“能测对象”分开：

```text
对象存在，不等于接口已接入。
```

## 五 · 有法：测量必须有问法

方法边界问：

```text
测量协议是否已指定？
```

Lean 中：

```lean
inductive ProtocolRegion : Type
  | specified
  | unspecified
```

主证明：

```lean
theorem protocol_boundary_complete :
    (∀ r : ProtocolRegion, ProtocolSpecified r ∨ ProtocolUnspecified r)
    ∧ (∀ r : ProtocolRegion, ProtocolSpecified r → ¬ ProtocolUnspecified r)
    ∧ (∀ r : ProtocolRegion, ProtocolUnspecified r → ¬ ProtocolSpecified r)
```

在量子文档里，这对应“测量基 / 问法”：

```text
问法决定什么能作为结果出现。
```

在统计文档里，这对应实验设计、采样规则、假设检验规则。没有协议，就没有可复核的测。

## 六 · 已校：无参照，不成测

校准不是附属步骤。没有校准，读数可能仍然出现，但它不具备本文所说的完整测量资格。

## 九 · 完整测量流程

Lean 中把六条轴合成一个最薄流程：

```lean
structure CeSkeleton where
  observability : ObservabilityRegion
  interface : InterfaceRegion
  protocol : ProtocolRegion
  calibration : CalibrationRegion
  readout : ReadoutRegion
  record : RecordRegion
```

完整测量定义为六项全在内侧：

```lean
def CompleteCe (m : CeSkeleton) : Prop :=
  Observable m.observability
    ∧ Interfaced m.interface
    ∧ ProtocolSpecified m.protocol
    ∧ Calibrated m.calibration
    ∧ Readable m.readout
    ∧ Recorded m.record
```

再把它映射回“量”的 measurement 边界：

```lean
def ceAsLiangMeasurement (m : CeSkeleton) : MeasurementRegion :=
  match m.observability, m.interface, m.protocol, m.calibration, m.readout, m.record with
  | .observable, .interfaced, .specified, .calibrated, .readable, .recorded =>
      .measurableByInterface
  | _, _, _, _, _, _ =>
      .unmeasurableByInterface
```

主证明：

```lean
theorem complete_ce_maps_to_liang_measurable (m : CeSkeleton) :
    CompleteCe m → MeasurableByInterface (ceAsLiangMeasurement m)

theorem incomplete_ce_maps_to_liang_unmeasurable
    (m : CeSkeleton)
    (h : ¬ CompleteCe m) :
    ceAsLiangMeasurement m = .unmeasurableByInterface
```

读法：

| 流程状态 | 落点 |
|---|---|
| 六项全齐 | “量”的 measurement 内侧 |
| 缺任一项 | “量”的 measurement 外侧 |

这就是本文对“完整测”的机器可检验核心。

## 十 · 测止于记录，推另起

测量产物和推断产物不同：

```lean
inductive CeProduct : Type
  | measurementRecord
  | inferenceClaim

theorem measurement_record_ne_inference_claim :
    CeProduct.measurementRecord ≠ CeProduct.inferenceClaim
```

这条边界很实用。

| 层 | 产物 |
|---|---|
| 测 | 读数 / 记录 |
| 推 | 模型、参数、假设、因果判断 |

统计文档中有链：

```text
元 ─→ 事 ─→ 域 ─→ 度 ─→ 率 ─→ 期 ─→ 测 ─→ 推
```

本文只完成“测”。“推”要进入统计推断、贝叶斯更新、假设检验或因果推断，不由测量记录本身自动给出。

## 十一 · 主定理

本文的总入口：

```lean
theorem ce_boundary_summary :
    (CeAxis.observability ≠ CeAxis.interface
      ∧ CeAxis.observability ≠ CeAxis.protocol
      ∧ CeAxis.observability ≠ CeAxis.calibration
      ∧ CeAxis.observability ≠ CeAxis.readout
      ∧ CeAxis.observability ≠ CeAxis.record
      ∧ ...)
    ∧ (∀ r : ObservabilityRegion, Observable r ∨ Unobservable r)
    ∧ ...
    ∧ (∀ m : CeSkeleton, CompleteCe m → MeasurableByInterface (ceAsLiangMeasurement m))
    ∧ (∀ m : CeSkeleton, ¬ CompleteCe m → ceAsLiangMeasurement m = .unmeasurableByInterface)
    ∧ CeProduct.measurementRecord ≠ CeProduct.inferenceClaim
    ∧ (∀ o : CollapseOutcome, Readable (collapseOutcomeReadout o))
```

总读法：

1. 六条“测”轴不同；
2. 六条轴各自内外穷尽且互斥；
3. 完整测量流程落到“量”的 measurement 内侧；
4. 缺任一项的流程落到 measurement 外侧；
5. 测量记录不是推断命题；
6. 量子时空 skeleton 的 collapse outcome 已经是可读输出。

## 十二 · 结论

“测”的闭合不是说所有经验问题都已解决，而是把测量作为流程边界固定下来：

```text
测
├─ 可观：对象进入观测窗口
├─ 可接：接口 / 仪器接上对象
├─ 有法：协议 / 问法已指定
├─ 已校：尺度 / 零点 / 参照已固定
├─ 可读：产生可读输出
└─ 可记：进入可复核记录
```

测量完整，才进入“量”的 measurement 内侧。
测量记录形成，仍不等于推断完成。
