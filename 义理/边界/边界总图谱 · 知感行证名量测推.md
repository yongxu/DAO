## 〇 · 边界声明

这张图谱不是世界本体分类，也不是统一场论。它只做一个可审计的结构判断：

```text
这些边界可以互相接续，但不能互相替代。
```

| 误混 | 为什么不成立 |
|---|---|
| 名 = 知 | 有指称不等于已明辨述贯存 |
| 知 = 证 | 可描述为理不等于有 proof certificate |
| 感 = 测 | 感可无仪器、校准、记录 |
| 行 = 知 | 知道不等于能做 |
| 测 = 量 | 测是读数流程；量是数量、度量、测度、测量、量词接口 |
| 量 = 推 | 量给值 / 尺度 / 测度；推给结论 |
| 推 = 证 | 推得结论不等于形成可复核证书 |

## 一 · 八界表

| 界 | 核心问题 | 完整条件 | 产物 / 落点 |
|---|---|---|---|
| 知 | 是否进入可描述之理 | 明、辨、述、贯、存 | `Knowable` |
| 感 | 是否进入感受接口 | 触、受、感、现、报 | `Feelable` |
| 行 | 是否进入行动接口 | 主体、能力、境况、操作、效果 | `Actionable` |
| 证 | 是否进入证明接口 | 命题、系统、推导、检查器、证书 | `Provable` |
| 名 | 是否进入指称接口 | 所指、符号、约定、域、消歧 | `Nameable` |
| 量 | 是否进入量化接口 | 数量、度量、测度、测量、量词 | quantity / metric / measure / measurement / quantifier |
| 测 | 是否形成可复核测量 | 可观、可接、有法、已校、可读、可记 | measurement record |
| 推 | 是否形成有担保结论 | 前提、规则、推导、结论、担保 | inference claim |

## 二 · 不是线性阶梯

这八界不是固定顺序的流水线。现实中它们会互相调用：

```text
名可以给知提供入口
感可以给测提供材料
测可以给量提供读数
量可以给推提供前提
推可以给证提供候选结构
证可以反过来改写知
行可以检验知、测、推的落地性
```

但“调用”不是“同一”。

一个测量记录可以被量化，但记录不是量本身。
一个推断可以被证明，但推断不是证明本身。
一个名字可以导向知识，但名字不是知识本身。

## 三 · Lean 中的总图谱

Lean 先把八界列成构造子：

```lean
inductive BoundaryLayer : Type
  | zhi
  | gan
  | xing
  | zheng
  | ming
  | liang
  | ce
  | tui
```

主证明是两两不同：

```lean
theorem boundary_layers_pairwise_distinct : ...
```

链式读法：

```lean
theorem boundary_layer_chain_distinct :
    BoundaryLayer.zhi ≠ BoundaryLayer.gan
    ∧ BoundaryLayer.gan ≠ BoundaryLayer.xing
    ∧ BoundaryLayer.xing ≠ BoundaryLayer.zheng
    ∧ BoundaryLayer.zheng ≠ BoundaryLayer.ming
    ∧ BoundaryLayer.ming ≠ BoundaryLayer.liang
    ∧ BoundaryLayer.liang ≠ BoundaryLayer.ce
    ∧ BoundaryLayer.ce ≠ BoundaryLayer.tui
```

这给出最低限度的形式保证：在本层中，八界不被编码成同一个东西。

## 四 · 读法

总图谱的实用读法很简单：

1. 先问当前问题落在哪个界；
2. 再检查该界的完成轴；
3. 若要跨界，明确写出转换关系；
4. 不用一个界的完成条件冒充另一个界的完成条件。

这能避免几类常见错误：

| 错误 | 更正 |
|---|---|
| “有名字，所以知道了” | 只能说明名界可能成立，还要检查知界 |
| “感到了，所以测到了” | 感界可能成立，测界还需观、接、法、校、读、记 |
| “测到了，所以有理论结论” | 测界只给记录，推界还要前提、规则、推导、结论、担保 |
| “推出来了，所以证明了” | 推界给 claim，证界还要系统、检查器、证书 |
| “证明了，所以能做” | 证界成立，不自动给行界的主体、能力、境况、操作、效果 |
