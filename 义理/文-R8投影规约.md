## 〇 · 边界声明

本文不主张：

| 不主张 | 原因 |
|---|---|
| R8 已无损容纳完整自然语言语义 | 自然语言有语境、历史、递归解释和外部指称 |
| 任意句子天然落在 R8 | 必须先给出 `Sentence → WenNormalForm → R8Semantics` 的解释器 |
| 高于 R8 的语义可无损塞回 R8 | 高维对象只能投影到 R8，并必须记录 `ProjectionLoss` |
| axis 是否恰当可由本文自动发现 | 本文给 certificate interface；具体独立性与语义负载由解释器或后续证明提供 |

本文主张：

| 主张 | 精确读法 |
|---|---|
| 文句可投影 | 给定解释器后，句子有 R8 可见坐标 |
| 归道可判为结构谓词 | 可见坐标为 `R8.origin` 即 `returnsDao` |
| 组合归道可用 XOR 路径表达 | `foldMasks [c, c] = origin` 是最小见证 |
| 高维可投影但有损 | `ProjectionLoss` 明确记录 `extraAxis` / `history` / `context` / `recursion` 等损失 |
| 升维恰当性可被审计 | `AxisAppropriate` 要求非零、独立证书、可投影 |

## 一 · 规约总式

本文的核心流水线是：

```text
Sentence
  → WenNormalForm
  → R8Semantics
  → visibleR8 : R8
```

Lean 中写作：

```lean
structure WenR8Interpreter where
  rewrite : Sentence → WenNormalForm
  interpret : WenNormalForm → R8Semantics

def sentenceProjection (I : WenR8Interpreter) (s : Sentence) : R8 :=
  visibleR8 (interpretSentence I s)
```

读法：

| 阶段 | 作用 |
|---|---|
| `Sentence` | 原始有限句子，只作为输入表面 |
| `WenNormalForm` | 改写后的“文”，是解释器的中间层 |
| `R8Semantics` | R8 可见语义形态 |
| `visibleR8` | 落在八轴闭合内的坐标 |

所以“任意句子都可检测在哪里”要读成：

> 对任意句子，只要解释器给出 `rewrite` 与 `interpret`，它就有一个 R8 可见投影。

机器入口：

```lean
theorem sentence_has_r8_projection (I : WenR8Interpreter) (s : Sentence) :
    ∃ c : R8, sentenceProjection I s = c
```

这不是完整自然语言理解，而是规约接口完整。

## 二 · R8Semantics 五类

一个文句进入 R8，不必只落成单个 cell。它可以是五种形态：

| 形态 | Lean 构造子 | 含义 |
|---|---|---|
| 点 | `cell c` | 句义压成一个 R8 cell |
| 算子 | `operator f` | 句义是一个 R8 endomap，以 `f origin` 作可见诊断点 |
| 路径 | `path masks` | 句义是一串 XOR masks |
| 区域 | `slice anchor pred` | 句义是 R8 中一个谓词区域，并给一个 anchor |
| 高维投影 | `projected c loss` | 句义高于 R8，只保留可见坐标和损失账本 |

对应定义：

```lean
inductive R8Semantics where
  | cell : R8 → R8Semantics
  | operator : (R8 → R8) → R8Semantics
  | path : List R8 → R8Semantics
  | slice : R8 → (R8 → Prop) → R8Semantics
  | projected : R8 → ProjectionLoss → R8Semantics
```

这样可以避免一个常见误读：不是所有句子都应被强行压成一个点。有些句子天然是操作、有些是路径、有些是区域判断、有些只是高维对象在 R8 下的影子。

## 三 · 归道

在 R8 内，归道的最小定义是：

```lean
def returnsDao (s : R8Semantics) : Prop :=
  visibleR8 s = R8.origin
```

读法：

```text
归道 = 可见 R8 坐标回到 origin / 道 / no-op anchor
```

对于 XOR path，组合方式是：

```lean
def foldMasks : List R8 → R8
  | [] => R8.origin
  | c :: rest => R8.xor c (foldMasks rest)
```

已经证明两个最小事实：

```lean
theorem empty_path_returnsDao :
    returnsDao (.path [])

theorem duplicated_mask_path_returnsDao (c : R8) :
    returnsDao (.path [c, c])
```

义理读法：

| 形式 | 读法 |
|---|---|
| 空路径归道 | 什么也不做就是道 |
| `c ⊕ c = 0` | 同一差异开两次，回到道 |
| `op₁ ⊕ ... ⊕ opₙ = 0` | 一组差异组合后相消 |

这接住“真可假，假可真”：每个二值 axis 的 toggle 是 involution；翻两次即归原，组合为零即归道。

## 四 · 升维、降维与 axis 恰当性

升维不是简单加一个格子，而是创造一个信息 axis：

```text
R_n × Bool → R_{n+1}
```

降维则是忘却该 axis：

```text
R_{n+1} → R_n
```

本文把“axis 是否恰当”压成证书接口：

```lean
structure AxisSpec where
  mask : R8
  role : AxisRole
  independentFromPrior : Prop
  projectable : Bool

def AxisAppropriate (a : AxisSpec) : Prop :=
  a.mask ≠ R8.origin
    ∧ a.independentFromPrior
    ∧ a.projectable = true
```

三条条件的意义：

| 条件 | 含义 |
|---|---|
| `mask ≠ origin` | 新 axis 不是空差异 |
| `independentFromPrior` | 在所选 frame 中不是旧 axis 的重复 |
| `projectable = true` | 升上去以后知道如何降回去 |

`AxisRole` 只记录粗角色：

```lean
inductive AxisRole where
  | yuan
  | yao
  | yin
  | guo
  | truth
  | named
  | contextual
```

这对应你说的“升维需要加元/爻”。元与爻不是额外神秘对象，而是 axis 的角色分配：一个承担 carrier，一个承担 dimension。具体某一步谁为元、谁为爻，是 doctrine 选择；形式上要求它可被审计。

## 五 · 高于 R8 的投影

若句子语义超过 R8，例如：

| 超出项 | 为什么超出 |
|---|---|
| 第九个独立 axis | 不在 `R0..R8` 静态闭合内 |
| 历史列表 | 需要 `List R8` 或 BaguaTuring 状态 |
| 语境依赖 | 需要外部环境或解释状态 |
| 自解释递归 | 进入 WenyanSelfInterp / Universal route |
| 高阶算子 | 可能需要函数空间或 L-tower |

则只能写作：

```lean
R8Semantics.projected c loss
```

损失种类：

```lean
inductive ProjectionLoss where
  | none
  | extraAxis
  | history
  | context
  | recursion
  | higherOperator
```

这就是“即使高于 8，也可以 project 到 8”的精确边界：

> 可以投影，但不能伪称无损。R8 坐标说明“八轴闭合内看见了什么”；`ProjectionLoss` 说明“八轴闭合看不见什么”。

机器入口：

```lean
theorem sentence_has_loss_ledger (I : WenR8Interpreter) (s : Sentence) :
    ∃ loss : ProjectionLoss, sentenceProjectionLoss I s = loss
```

## 六 · R2 → R4 → R8 的尺度读法

平方塔给出另一条尺度规则：

```text
R1 × R1 = R2
R2 × R2 = R4
R4 × R4 = R8
```

本文把这条规则读成：

| 跃迁 | 规约读法 |
|---|---|
| `R1 → R2` | 第一次稳定二轴配对 |
| `R2 → R4` | 规则 1：一组作元，一组作爻 |
| `R4 → R8` | 规则 2：Mian 尺度再作一次元/爻配对 |

这与 incrementing 视角不冲突。incrementing 负责每次加一 bit；squaring 负责在特定尺度上说明“同一层与自身相配”。

因此：

```text
R8 = R4 × R4
```

可读为：

```text
一个 Mian 作元
一个 Mian 作爻
```

也可对偶反读。群结构本身无偏好，文义解释器必须声明自己采用哪一种。

## 七 · 总 theorem

本文总入口：

```lean
theorem wen_r8_projection_calculus_summary (I : WenR8Interpreter) :
    (∀ s : Sentence, ∃ c : R8, sentenceProjection I s = c)
    ∧ (∀ s : Sentence, ∃ loss : ProjectionLoss, sentenceProjectionLoss I s = loss)
    ∧ returnsDao (.cell R8.origin)
    ∧ (∀ c : R8, returnsDao (.path [c, c]))
    ∧ (∀ a : AxisSpec, AxisAppropriate a → a.projectable = true)
```

五个子句分别说：

| 子句 | 读法 |
|---|---|
| 每句有 R8 投影 | 给定解释器后可定位 |
| 每句有损失账本 | 高维投影不偷渡 |
| origin cell 归道 | 道为可见 anchor |
| 重复 mask path 归道 | 同一差异翻两次归零 |
| 恰当 axis 可投影 | 升维必须能说明降维 |

这就是“完整表达”的最小机器核心。

本文故意保守。后续若要加强，可按三步做：

| 强化 | 需要新增 |
|---|---|

这些都是加强解释器，不是修改本文规约。
