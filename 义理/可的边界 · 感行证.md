## 一 · “可”的轴

“可”不是一个单轴词。

| 轴 | 问题 | 典型误读 |
|---|---|---|
| 可知 | 能否作为可描述之理被知道 | 把知道等同于感到 |
| 可感 | 能否进入感受 / 现象接口 | 把感到等同于证明 |
| 可行 | 能否进入行动 / 实践接口 | 把可说等同于可做 |
| 可证 | 能否进入证明接口 | 把不可证等同于无意义 |

Lean 中先给轴：

```lean
inductive KeAxis : Type
  | know
  | sense
  | act
  | prove
```

并证明这些轴不能直接塌缩：

```lean
theorem ke_axes_pairwise_distinct :
    KeAxis.know ≠ KeAxis.sense
    ∧ KeAxis.know ≠ KeAxis.act
    ∧ KeAxis.know ≠ KeAxis.prove
    ∧ KeAxis.sense ≠ KeAxis.act
    ∧ KeAxis.sense ≠ KeAxis.prove
    ∧ KeAxis.act ≠ KeAxis.prove
```

这一步很关键：可感、可行、可证都和可知有关，但不是可知的别名。

---

## 二 · 可感 / 不可感

可感边界问的是：

```
某物是否进入感受 / 现象接口？
```

它不是问“能不能证明”，也不是问“能不能说清”。

Lean 中：

```lean
inductive SensibilityRegion : Type
  | feelable
  | unfeelable
```

定义：

```lean
def Feelable (r : SensibilityRegion) : Prop :=
  r = .feelable

def Unfeelable (r : SensibilityRegion) : Prop :=
  r = .unfeelable
```

证明：

```lean
theorem sensibility_boundary_complete :
    (∀ r : SensibilityRegion, Feelable r ∨ Unfeelable r)
    ∧ (∀ r : SensibilityRegion, Feelable r → ¬ Unfeelable r)
    ∧ (∀ r : SensibilityRegion, Unfeelable r → ¬ Feelable r)
```

读法：

| 命题 | 含义 |
|---|---|
| 穷尽 | 在当前感受接口中，只分可感 / 不可感 |
| 互斥 | 同一对象不能在同一接口下既可感又不可感 |

这只是感受边界的 typed skeleton。它不说明感受内容是什么，不说明 qualia，也不说明主体间验证。

---

## 三 · 可行 / 不可行

可行边界问的是：

```
某物是否能进入行动 / 实践接口？
```

可行不等于可知。

有些东西可知但不可行：

```
我知道某个天体轨道，但不能改变它。
```

有些东西可行但未必已经被完整说清：

```
身体技能、习惯动作、现场判断。
```

Lean 中：

```lean
inductive ActionabilityRegion : Type
  | actionable
  | unactionable
```

证明：

```lean
theorem actionability_boundary_complete :
    (∀ r : ActionabilityRegion, Actionable r ∨ Unactionable r)
    ∧ (∀ r : ActionabilityRegion, Actionable r → ¬ Unactionable r)
    ∧ (∀ r : ActionabilityRegion, Unactionable r → ¬ Actionable r)
```

读法：

| 命题 | 含义 |
|---|---|
| 穷尽 | 在当前行动接口中，只分可行 / 不可行 |
| 互斥 | 同一对象不能在同一接口下既可行又不可行 |

这同样不展开行动理论；只先把边界形式安放。

---

## 四 · 可证 / 不可证

可证边界与可知最接近，但仍不能混同。

可知边界说：

```
是否能作为可描述之理进入语言？
```

可证边界说：

```
是否能在当前 proof interface 中给出证明？
```

一个命题可以有意义、可说、可知，但在某个形式系统中不可证。

因此本文只做 typed skeleton：

```lean
inductive ProofRegion : Type
  | provable
  | unprovable
```

证明：

```lean
theorem proof_boundary_complete :
    (∀ r : ProofRegion, Provable r ∨ Unprovable r)
    ∧ (∀ r : ProofRegion, Provable r → ¬ Unprovable r)
    ∧ (∀ r : ProofRegion, Unprovable r → ¬ Provable r)
```

同时说明：这组 proof-boundary 的区分本身仍然可说：

```lean
def proofRegionAsKnowable (_r : ProofRegion) : KnowableRegion :=
  .withinDescribableLi

theorem proof_boundary_is_sayable_skeleton (r : ProofRegion) :
    Sayable (proofRegionAsKnowable r)
```

这不是说所有不可证命题都已被解决。恰恰相反：本文只证明“可证 / 不可证”这个边界可以被语言表达；不证明具体命题是否可证。

---

## 五 · 三组边界的关系

| 边界 | 内侧 | 外侧 | 与可知关系 |
|---|---|---|---|
| 可知 | 可描述之理 | 不可作为理说出 | 主轴 |
| 可感 | 可进入感受接口 | 不可进入感受接口 | 不等于可知 |
| 可行 | 可进入行动接口 | 不可进入行动接口 | 不等于可知 |
| 可证 | 可在 proof interface 内证明 | 不可在该 interface 内证明 | 用可知语言表达，但不等于可知 |

所以：

```
可知 ≠ 可感 ≠ 可行 ≠ 可证
```

更准确：

```
它们是四个不同接口上的“可”。
```

## 七 · 主定理

```lean
theorem ke_boundary_summary :
    (KeAxis.know ≠ KeAxis.sense
      ∧ KeAxis.know ≠ KeAxis.act
      ∧ KeAxis.know ≠ KeAxis.prove
      ∧ KeAxis.sense ≠ KeAxis.act
      ∧ KeAxis.sense ≠ KeAxis.prove
      ∧ KeAxis.act ≠ KeAxis.prove)
    ∧ (∀ b : KeBoundary, b.side = .inside ∨ b.side = .outside)
    ∧ (∀ r : SensibilityRegion, Feelable r ∨ Unfeelable r)
    ∧ (∀ r : SensibilityRegion, Feelable r → ¬ Unfeelable r)
    ∧ (∀ r : SensibilityRegion, Unfeelable r → ¬ Feelable r)
    ∧ (∀ r : ActionabilityRegion, Actionable r ∨ Unactionable r)
    ∧ (∀ r : ActionabilityRegion, Actionable r → ¬ Unactionable r)
    ∧ (∀ r : ActionabilityRegion, Unactionable r → ¬ Actionable r)
    ∧ (∀ r : ProofRegion, Provable r ∨ Unprovable r)
    ∧ (∀ r : ProofRegion, Provable r → ¬ Unprovable r)
    ∧ (∀ r : ProofRegion, Unprovable r → ¬ Provable r)
    ∧ (∀ r : ProofRegion, Sayable (proofRegionAsKnowable r))
    ∧ (∀ b : DeferredKeBoundary, handledHere b = false ∧ handledInKeBoundary b = true)
```

这就是本文完成的最小闭包。

## 八 · 结论

前文已经给出：

```
可知 = 可说 = 可描述之理
```

本文补齐：

```
可感 / 不可感
可行 / 不可行
可证 / 不可证
```

但补齐不等于混同。

最终结构是：

```
可
├─ 可知：语言可描述之理
├─ 可感：能否进入感受接口
├─ 可行：能否进入行动接口
└─ 可证：能否进入当前 proof interface
```

其中可证最接近可知，但也只是“用可知语言表达的证明边界”，不是可知本身。
