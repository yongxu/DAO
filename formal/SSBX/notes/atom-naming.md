## 仍需义位拆分的 27 字

这些字已注册为 `needsSenseSplitAtoms`：可回根，但不能把语义压成一个无歧义定义。

```text
生、开、闭、正、邪、真、道、人、天、子、之、意、识、心、焦、聚、模、理、法、面、气、神、精、性、礼、信
```

下一步若做语义充分证明，应补：

```text
生1 / 生2 / 生3
之1 / 之2 / 之3
天1 / 天2 / 天3
子1 / 子2
```

当前阶段只证明「它们都有核心归属和回根路径」，不证明具体语境中必须取哪一个义项。

## 新增三相转换字

为间—物—动三相转换补入：

| 类别 | 字 |
|---|---|
| 单字 | 露、隙、发、凝 |
| 生成项 | 成界、露隙、生合、生新隙、成态 |

对应：

```text
间 → 物：聚、显、成界
物 → 间：分、开、露隙
间 → 动：续、生合、发
动 → 间：转、岐、生新隙
物 → 动：行、应、变
动 → 物：积、凝、成态
```

为 v10 正名融贯层补入：

| 类别 | 字 |
|---|---|
| 单字 | 系、耦、剖 |
| 生成项 | 开法、开剖、自似 |

- `系` 接回关系域
- `耦` 接回间与间的相互启阻
- `剖` 接回开剖
- `开法` 与既有 `史法` 合成法之双源
- `自似` 用于尺度防盲

## 暂不整合为单字派生的项

保留为模型接口或待校项：

- **五个核心原始接口**：`域、格、权、生、校`
- **派生接口记号**：`I_F、K_Γ、w_Γ、D_KΓ、norm_KΓ、oplus_F、κ、κ_star、Ω、Ω_B、μ、Pr、Φ_形、Φ_因、Φ_结、Π_开、Q_audit、Θ、Attention`
- **待校项**：`审校数据、度期计算、开势投影、正邪阈值、经验校准、邪续`

派生接口记号以单字算子组合定义，可用单字算子为：`域、格、权、生、校、形、因、结`。其中 `域、格、权、生、校` 是五个核心原始接口；`形、因、结` 是已入籍单字元，可作为审校接口的单字算子，不因此升格为模型原始接口。

### 接口记号 → 单字 dispatch 映射

```text
I_F        := 域
K_Γ        := 格
w_Γ        := 权
D_KΓ       := 格权
norm_KΓ    := 权格
oplus_F    := 生
κ          := 格权
κ_star     := 格权校
Ω          := 权
Ω_B        := 权校
μ          := 权
Pr         := 权校
Φ_形       := 校形
Φ_因       := 校因
Φ_结       := 校结
Π_开       := 校权
Q_audit    := 校
Θ          := 校权
Attention  := 权
```

原因：这些接口需要模型、度量、阈值、经验审校或外部理论接口，不能仅由汉字派生获得语义充分性。旧记号可保留为实现层 API 或公式缩写，但 Lean 名册中的原始项只剩五个入口。

## 后续可考虑的和谐映射

- **形 / 型**：建议区分——`形` 指对象或过程自身所显之模式，`型` 指模型中抽取出的型式、类或 schema。若纳入名册，可先试 `型 := 模形` 或 `型 := 格形`。
- **性**：字形上可读作 `心 + 生`。当前 `性` 已是人行类单字元；若做派生说明，可记为 `性 ≈ 心生`（心之可生倾向）。适合作为训诂/语义映射，不宜贸然替换为正式生成项，除非重整人行类根系。
