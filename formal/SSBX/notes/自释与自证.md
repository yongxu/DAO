# 自释 与 自证 · 分层

> **Status**: 分层文档，跨整个「文自编译自证」路线。
> **Risk mitigated**: 路径丙 § 风险 4（「自释」与「自证」混淆）。
> **Locus**: 已有自释见 `WenyanSelfInterp.lean`；自证待建于 M3-甲 `WenyanKernel.lean`。

---

## 引

「文证文」一语，实有两层。混淆此二者，则路径丙之 M2 → M3 转折点不可辨。

| 维 | 自释 (semantic) | 自证 (syntactic) |
|---|---|---|
| 输入 | 程序 + 状态 | 程序源（List Cell192） |
| 输出 | 状态' | Bool（合 / 不合） |
| 关心 | **执行等价**：metaStep 与 step 在 cur 上一致 | **结构合度**：jump target 不越界、tag 在 12 字内、参数齐全 |
| 触 Gödel | **否**——仅模拟单步 | **否（若有界）**——只判 well-formedness，不判停机 |
| 已实 | `WenyanSelfInterp.lean §5-6`（13 子句已证） | 待建（M3-甲） |

## 一 · 自释（自我解释）

**定义**：「程之释由程书就」——一段程序作为数据被另一段程序解释执行，得到与原解释一致的结果。

**形式签名**（已存在）：
```lean
def metaStep : YiState → List Cell192        -- WenyanSelfInterp §5
theorem metaStep_cur_correct (s : YiState) :
    (metaStep s).head? = some s.step.cur     -- WenyanSelfInterp §6
```

**所证**：编码后的「下一状态之 cur」等于直接 step 的 cur。这是**单步模拟**，足以建立编码-执行同构（13 子句公示）。

**关键性质**：
- 编码-解码 round-trip 全数（除三条 Nat-参数指令在 P17 中收尾）
- 单步语义保形（cur 域已证；history/pc/halted 在 P17 末统一）
- 不涉及对**长度未知**程序的判定——故不撞 Gödel

**类比**：Lean 的 `Expr` 是 Lean term 的语义编码；Lean 中可写 `evalExpr`，是「Lean 释 Lean」之自释面。

## 二 · 自证（自我证明 / 自我审查）

**定义**：「程之合法由程审就」——一段验证程序读入另一段程序的源代码，判断其是否符合 baguaWen 受控语法，且能验证自身的源。

**形式签名**（待建）：
```lean
def «判型良» : List YiInstr → Bool                 -- M3-甲 / WenyanKernel.lean
theorem «微核自验» :
    «判型良» «微核源» = true ∧
    ProgEnc.decInstrs «微核数» = some («微核源», []) := by native_decide
```

**所证**：句法层不动点——文之 verifier 接受文之 verifier 的源。

**关键性质**：
- **Bool-valued** 总判定，可 `native_decide`
- 不判停机；只判：
  - 12 字 tag 之合法性（`tag.val < 12`）
  - jump target 不越界（`t < program.length`）
  - 时态 / 爻位参数在 Fin 范围内
  - 数词在 1..64
- **有界**——输入是有限的 List YiInstr，输出在常数时间内可决

**类比**：Lean 的 kernel typechecker 在 `Expr` 上做的就是这件事——句法结构合度判定。Lean kernel 不判 `Halts`，自证亦不判。

## 三 · 二者关系

```
┌──────────────┐        ┌──────────────┐
│   自释        │        │   自证        │
│ (semantic)    │        │ (syntactic)   │
├──────────────┤        ├──────────────┤
│ metaStep      │  ←→   │ «判型良»      │
│ runFuel       │        │ ProgEnc.decInstrs │
│ 13 子句       │        │ «微核自验»    │
└──────────────┘        └──────────────┘
       ↓                        ↓
   M2 之 PoC              M4-甲 之自宿主
   (运行等价)              (源被自身接受)
```

**M2 PoC 同时见证两层**：
1. `WenyanParser.«解程» «道判源» = some daoJudgeProg`（自证之雏形：源 → 形式项）
2. `wenEval (encProg daoJudgeProg) 10 ≠ none`（自释之实证：形式项 → 执行）

**M4 闭合两层**：
1. `«判型良» «微核源» = true`（自证完整化：源被自身接受）
2. `metaStep`（已有）作为「微核源」内部 dispatch 的语义靶——保证「自证之程」依「自释之程」运行

## 四 · 边界声明

**绝不声称**：
- `∀ p, Decidable (∃ n, halts p n)`——撞 Gödel，不可
- `∀ p, «判型良» p = true → halts p`——自证不蕴含停机
- `自证 = 自释`——二者层级不同

**确实声称**（在 M0-M5 完成后）：
- 自释：编码 round-trip + bounded 单步模拟一致
- 自证：bounded well-formedness check + self-acceptance
- 两者通过 `«微核源»` 这一具体 List YiInstr 在 M4 处会合

## 五 · 命名约定（写代码时遵守）

| 字 / 词 | 限定层 |
|---|---|
| `metaStep`、`encInstr`、`decInstr`、`encProg`、`encState`、`runFuel` | **自释** |
| `«判型良»`、`«判停»`、`«验程»`、`«微核源»`、`«微核数»`、`«微核自验»` | **自证** |
| `«文之解析»` (parser)、`«文之印程»` (printer) | **句法层**（自证之前置） |
| `wenEval` (semantic evaluator) | **自释**（M2 之主产出） |

混用即为命名错误——code review 时 reviewer 应以此表反查。
