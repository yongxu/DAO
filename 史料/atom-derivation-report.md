# 单字整合报告

## 当前结论

当前 Lean 体系已经不再让 221 个登记单字直接作为单根图中的近根 source。

新的单根路径是：

```text
一元
  -> 面
    -> 核心单字 CoreAtom
      -> 登记单字 AtomName
        -> 正式项 / 构造阶段 / claim
```

`CoreAtom` 当前为 43 个：

```text
一、元、之、法、行、成、序、物、场、形、动、变、生、续、开、闭、审、校、证、真、正、邪、夺、共、仁、道、模、度、期、算、理、心、聚、焦、意、识、注、人、做、齐、控、天、子
```

Lean 证明：

```lean
all_core_atoms_reachable_from_root
all_atoms_have_core
all_atoms_return_through_core
all_atoms_pass_through_core
```

## 已经可以整合的字

这些字已经由 `atomCore` 收束为核心单字，作为派生单字处理：

- 文法类：`显、未、尽、间、可、候、新、合、悖、入、待、及、外、为、而、露、隙` 回 `法`，`序` 回 `序`，`行` 回 `行`，`成` 回 `成`。
- 物面类：`境、系、身、态、状、相、因、结、据、迹、史、积、精、气、神、几、临、岐、扰、伤、散、坍、径、返、自、替、基、线、域、格、世、界、对、底、凝` 回 `物`。
- 生面类：`达、维、应、限、暂、稳、展、通、流、和、平、护、存、能、息、再、关、冻、修、复、转、发` 回 `生`，`绝、断` 回 `闭`。
- 审校类：`异、众、互、受、独、查、源、执、著、黜、蔽、程、败、伪、似、实、回、观、照` 回 `审`，`验` 回 `校`，`归` 回 `证`。
- 价值类：`危、好、坏、由、繁、荣、义、善` 回 `正`，`依、压、偏、筛、放、抑、汰、益、损、险、迫` 回 `夺`，`己、同` 回 `共`。
- 模型类：`面、评、价、础、科、学、权、重、差、势、强、向、机、率、阈、效、责、耦、剖` 回 `模`。
- 证明/理性类：`辨、知、智、轨、逻、辑、构、造、纳、论、普、遍、明` 回 `理`，`演` 回 `算`。
- 心智/注意类：`感、择、情` 回 `心`，`图` 回 `意`，`调、门、分、配、持、竞、争、记、忆、上、下、工` 回 `注`。
- 人行类：`礼、信、性、目、标、制、层` 回 `人`，`作` 回 `做`。

## 仍需义项拆分的字

这些字可以回根，但不能把语义压成一个无歧义定义。它们在 `AtomDerivation.lean` 中登记为 `needsSenseSplitAtoms`：

```text
生、开、闭、正、邪、真、道、人、天、子、之、意、识、心、焦、聚、模、理、法、面、气、神、精、性、礼、信
```

下一步如果要做语义充分证明，应为这些字补：

```text
生1 / 生2 / 生3
之1 / 之2 / 之3
天1 / 天2 / 天3
子1 / 子2
```

当前阶段只证明“它们都有核心归属和回根路径”，不证明具体语境中必须取哪一个义项。

## 新增三相转换词

本轮补入三相转换所需单字与生成项：

```text
单字：露、隙、发、凝
生成项：成界、露隙、生合、生新隙、成态
```

它们对应：

```text
间 → 物：聚、显、成界
物 → 间：分、开、露隙
间 → 动：续、生合、发
动 → 间：转、岐、生新隙
物 → 动：行、应、变
动 → 物：积、凝、成态
```

本轮继续补入 v10 正名融贯层所需字与生成项：

```text
单字：系、耦、剖
生成项：开法、开剖、自似
```

其中 `系` 接回关系域，`耦` 接回间与间的相互启阻，`剖` 接回开剖；`开法` 与既有 `史法` 合成法之双源，`自似` 用于尺度防盲。

## 暂不应整合为文字派生的项

以下项仍应保留为模型接口或待校项：

- 核心原始接口：`域、格、权、生、校`。
- 派生接口记号：`I_F、K_Γ、w_Γ、D_KΓ、norm_KΓ、oplus_F、κ、κ_star、Ω、Ω_B、μ、Pr、Φ_形、Φ_因、Φ_结、Π_开、Q_audit、Θ、Attention`。
- 待校项：`审校数据、度期计算、开势投影、正邪阈值、经验校准、邪续`。

派生接口记号以单字算子组合定义。右侧可用单字算子先定为：

```text
域、格、权、生、校、形、因、结
```

其中 `域、格、权、生、校` 是五个核心原始接口；`形、因、结` 是已入籍单字元，可作为审校接口的单字算子，不因此升格为模型原始接口。

```text
I_F      := 域
K_Γ      := 格
w_Γ      := 权
D_KΓ     := 格权
norm_KΓ  := 权格
oplus_F  := 生
κ        := 格权
κ_star   := 格权校
Ω        := 权
Ω_B      := 权校
μ        := 权
Pr       := 权校
Φ_形     := 校形
Φ_因     := 校因
Φ_结     := 校结
Π_开     := 校权
Q_audit  := 校
Θ        := 校权
Attention := 权
```

两个后续可考虑的和谐映射：

- `形 / 型`：建议区分为 `形` 指对象或过程自身所显之模式，`型` 指模型中抽取出的型式、类或 schema。若纳入名册，可先试 `型 := 模形` 或 `型 := 格形`。
- `性`：字形上可读作 `心 + 生`。当前 `性` 已是人行类单字元；若做派生说明，可记为 `性 ≈ 心生`，即心之可生倾向。它适合作为训诂/语义映射，不宜贸然替换为正式生成项，除非重整人行类根系。

原因：这些接口需要模型、度量、阈值、经验审校或外部理论接口，不能仅由汉字派生获得语义充分性。旧记号可保留为实现层 API 或公式缩写，但 Lean 名册中的原始项只剩五个入口。
