## 三 · 关键定理详释

### 3.1 § 1：基础非平凡性

**`halts_daoJudgeProg`**：5 指令的道判机在任何 Hexagram 上必停。
**形式**：`∀ h : Hexagram, Halts daoJudgeProg h`
**证明**：直接由 `daoJudgeProg_total_within_10` 之 Halts 形式提升（fuel = 10 见证）。

**`loopProg_not_halts`**：单条 `jump 0` 之程序永不停。
**形式**：`∀ h : Hexagram, ¬ Halts loopProg h`
**证明**：核心引理 `loopProg_step_self : (init h loopProg).step = init h loopProg`（一步即 idem）+ Nat 归纳推 `runFuel n` 始终不停。

### 3.2 § 2：有限燃料不完备 ★（无公理）

**主定理 `li_incomplete_finite`**：无均一燃料界 N 可判 Halts。
**形式**：
```
¬ ∃ N : Nat, ∀ (P : List YiInstr) (h : Hexagram),
    Halts P h ↔ ((YiState.init h P).runFuel N).halted = true
```

**反例构造**：`slowProg N := List.replicate N nop ++ [halt]`（即 N 个 nop 后接 halt）。
- 必停（fuel = N+1 见证）；
- 但 `runFuel N` 仅执行 N 步 nop，pc 至 N 而尚未执行 halt，故 `halted = false`。

**关键引理 `runFuel_succ_right`**：`s.runFuel (n+1) = (s.runFuel n).step`。
**证明**：双重归纳 + 分情形 (s.halted = true / false)；约 30 行。
**用途**：使 N 步演化可由「N 次 step」累积。

**`slowProg_state_after`**：从 `init h (slowProg N)` 出发，对任 k ≤ N，`runFuel k` 后状态恰是 `pc=k, halted=false, prog=slowProg N`。
**证明**：归纳 k；用 `runFuel_succ_right` + `step_nop` + `slowProg_get_lt`。

> **意义**：此定理**完全不依赖任何外部公理**（包括 Kleene），是真正的 Gödel-flavored incompleteness——「理之停机不被任何 finite-fuel decision procedure 封闭」。

### 3.3 § 3-5：Kleene + 主定理 ★

**`KleeneInverter` 性质**：
```lean
def KleeneInverter : Prop :=
  ∀ (decide : List YiInstr → Hexagram → Bool),
    ∃ D : List YiInstr, ∀ h : Hexagram, Halts D h ↔ decide D h = false
```

**说明**：对任 Lean Bool decision function `decide`，存在 YiProg `D` 使 D 在 h 上停机当且仅当 `decide(D, h) = false`。即 D 之停机性「反相」decide 在 (D, h) 上的判断。

**形式上**：这是 **Kleene 递归定理对 BaguaTuring 之实例**（Church-Turing 论题落位）。完全形式化需在 YiInstr 内构造 quine（机械约 500 行工程，留 future work）。

**条件主定理 `halts_undecidable_under_kleene`**：
```
KleeneInverter →
¬ ∃ decide : List YiInstr → Hexagram → Bool,
    ∀ P h, decide P h = true ↔ Halts P h
```

**证明**：标准对角线。
1. 设 `decide` 判 Halts。
2. 由 `KleeneInverter` 取 `D` 使 `Halts D h ↔ decide D h = false`。
3. 又 `decide D h = true ↔ Halts D h`。
4. 链式：`decide D h = true ↔ decide D h = false`。
5. 在 Hexagram.qian 上分情形 (`decide D qian = true / false`) 皆得矛盾。

**唯一公理 `kleene_recursion_axiom : KleeneInverter`**：
- 类型为 `Prop`；
- 不引入新数据；
- 捕获 Church-Turing 论题对 BaguaTuring 之断言。

**无条件版 `halts_undecidable_internally`**：
- 直接由 `halts_undecidable_under_kleene kleene_recursion_axiom` 得。

### 3.4 § 6：衍生推论

**`not_halts_undecidable_under_kleene`**：「`¬ Halts P h`」也不可判。
**证明**：`KleeneInverter` 应用于 `(fun P h => !decide_neg P h)`，链式得 `Halts D h ↔ ¬ Halts D h`，矛盾。

**`halts_at_fixed_undecidable_under_kleene`**：固定任意 `h₀ : Hexagram`，「`P` 是否在 `h₀` 上停机」也不可判。
**证明**：`KleeneInverter` 应用于忽略 h 的 `(fun P _ => decide_h P)`，得 `Halts D h₀ ↔ decide_h D = false`，与 spec 联立矛盾。

### 3.5 § 7：Rice 四象齐备 ★

四象之形式 / 道家对应 / Lean 名：

| 四象 | 卦象 | 形式 | 不可判定理（条件版 / 无条件版） |
|---|---|---|---|
| **太阳** | ☰☰（处处停） | `∀ h, Halts P h` | `halts_on_all_undecidable_under_kleene` / `halts_on_all_undecidable` |
| **少阳** | ☰☷（有处停） | `∃ h, Halts P h` | `halts_on_some_undecidable_under_kleene` / `halts_on_some_undecidable` |
| **少阴** | ☷☰（有处不停） | `∃ h, ¬ Halts P h` | `halts_on_some_no_undecidable_under_kleene` / `halts_on_some_no_undecidable` |
| **太阴** | ☷☷（处处不停） | `∀ h, ¬ Halts P h` | `halts_on_none_undecidable_under_kleene` / `halts_on_none_undecidable` |

四象间存在 **De Morgan 对偶**：Π_all ↔ ¬ Π_some_no，Π_none ↔ ¬ Π_some。形式上：
- `halts_on_some_no_undecidable_under_kleene` 由 `halts_on_all_undecidable_under_kleene` 经布尔取反 (`fun P => !decide_some_no P`) 简化得到。
- `halts_on_some_undecidable_under_kleene` 与 `halts_on_none_undecidable_under_kleene` 各自独立用 KleeneInverter 走 `cases hd : decide_X D` 推矛盾。

**`halts_on_some_undecidable_under_kleene`**：「`∃ h, Halts P h`」不可判。
**证明**：取 KleeneInverter 之 D（用 `(fun P _ => decide_some P)`），分情形：
- decide_some D = true：spec 给 `∃ h, Halts D h`，但 Kleene 给 `∀ h, ¬ Halts D h`，矛盾。
- decide_some D = false：spec 给 `¬ ∃ h, Halts D h`，但 Kleene 给 `∀ h, Halts D h`，取 qian 即矛盾。

**`halts_on_none_undecidable_under_kleene`**：「`∀ h, ¬ Halts P h`」不可判。
**证明**：直接对 `(fun P _ => !decide_none P)` 应用 KleeneInverter。两情形均得矛盾。

**`halts_on_all_undecidable_under_kleene`**：「`∀ h, Halts P h`」不可判。
**证明**：用 `(fun P _ => decide_all P)` 应用 KleeneInverter，按 `decide_all D` 取真假分情：true 与 Kleene 不停冲突；false 与 Kleene 处处停冲突。

**`halts_on_some_no_undecidable_under_kleene`**：「`∃ h, ¬ Halts P h`」不可判。
**证明**：归约至 `halts_on_all_undecidable_under_kleene`：以 `fun P => !decide_some_no P` 为「`∀ h, Halts P h`」之候选 decider，由布尔取反 + De Morgan 即得。

**`rice_four_images_under_kleene`**（bundle）：以 4-tuple `⟨_, _, _, _⟩` 一并表出四象皆不可判。
**`rice_four_images`**：无条件版本，由 bundle 应用 `kleene_recursion_axiom` 直接得。

> **注**：这是 Rice 主定理之 **可证实例完整四象**——实际已涵盖 Halting 性质上「∀/∃ × halt/¬halt」的全部 4 个非平凡均一外延性质。完全的 Rice 主定理（任意非平凡外延性质皆不可判）仍需 program transformer，留 future work。

## 六 · 公理使用之诚实陈述

`kleene_recursion_axiom : KleeneInverter` 是本模块**唯一**的公理。

### 6.1 它是什么

形式：
```lean
axiom kleene_recursion_axiom :
  ∀ (decide : List YiInstr → Hexagram → Bool),
    ∃ D : List YiInstr, ∀ h : Hexagram, Halts D h ↔ decide D h = false
```

### 6.2 它为什么可信

- **数学事实**：Kleene 递归定理（quine 构造）在任何 Turing-complete 系统中皆成立。
- **本系统已 TC**：BaguaTuring §7b 论证 12-指令系统包含 universal computation 之四件套（无界跳转 / 无界程序长度 / 无界 history / 数据依赖分支）。
- **直接构造可行**：在 YiInstr 内显式构造 quine 是机械的工程，估约 500 行 Lean。

### 6.3 它能被去掉吗

**可以**——通过 §6.2 所述的 quine 显式构造。具体步骤：
1. 设计 `codeYiInstr : YiInstr → List Cell192` 编码（约 50 行）；
2. 实现 universal interpreter `universalProg : List YiInstr`（约 200 行）；
3. 构造 self-replicating quine `quineProg : List YiInstr`（约 100 行 + Kleene 双递归）；
4. 证 `KleeneInverter` 在此 quine 系统下成立（约 150 行）；

合计约 500 行，全机械工程，无新数学。本轮不实现。

### 6.4 公理之外没有其他依赖

- 无 Mathlib 依赖；
- 无 Classical 直接调用（除 `decide` 在 Bool 上之内禀使用）；
- 无 sorry。
